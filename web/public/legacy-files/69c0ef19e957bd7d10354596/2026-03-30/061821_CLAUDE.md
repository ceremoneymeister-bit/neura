## ДНК-правила (НЕУДАЛЯЕМЫЕ)

### 🔍 Скилл-чек (перед ЛЮБОЙ задачей)
Перед началом работы — 1 строка в ответе:
`🔍 Скилл: [название] → читаю SKILL.md` или `🔍 Скилл: не найден → выполню без, создать?`
Это обязательно. Без скилл-чека не начинать. Если подходит несколько — перечислить все.

### 📊 Капсулы — дашборд перед работой (НЕУДАЛЯЕМОЕ)
При ЛЮБОЙ задаче с капсулами, ботами пользователей, эталоном или деплоем:
1. **ДО работы:** `python3 scripts/capsule-dashboard.py` — убедись что всё active
2. **ПОСЛЕ работы:** повторить дашборд — убедись что ничего не сломалось
3. **Эталон:** при изменении капсулы → внести изменение и в `neura-capsule/` (чтобы новые капсулы получали фикс из коробки)
4. **Реестр:** при добавлении/удалении капсулы → обновить `capsule-profiles.json`, `capsule-dashboard.py`, `neura-bridge.py`
Без дашборда работа с капсулами НЕ начинается.

### 💾 Фиксация (после КАЖДОГО ответа)
1. SESSION_LOG.md — 1 строка что сделано + `💾`
2. Если создан/изменён клиент, капсула, бот, группа → обновить HANDOFF.md
3. Если новый факт о проекте/клиенте → обновить memory
Без `💾` ответ считается незавершённым.

### 🔄 Обновление скиллов (после КАЖДОЙ задачи, НЕУДАЛЯЕМОЕ)
После завершения задачи — проверь: использовал ли ты скилл?
1. Если да → **УЛУЧШИ** SKILL.md (не просто допиши — сделай лучше):
   - Нашёл баг → **исправь** секцию, которая привела к ошибке (не дописывай "урок")
   - Подход сработал лучше → **замени** старый подход на новый
   - Ограничение инструмента → **добавь** в антипаттерны или предусловия
   - Устаревшие инструкции → **удали** (не комментируй, не помечай deprecated)
   - Принцип: каждое использование скилла делает его ЛУЧШЕ, а не ДЛИННЕЕ
2. Если скилл НЕ использовался, но ДОЛЖЕН был → обнови триггеры в таблице маршрутизации
3. Подтверди: `**Скиллы обновлены:** [список]` или `**Скиллы:** без изменений`
Без этой строки задача считается незавершённой (как и без 💾).

### 📨 Telegram-отправка (НЕУДАЛЯЕМОЕ)
Ты МОЖЕШЬ отправлять сообщения через `python3 scripts/tg-send.py <user> "Текст"`.
Правила:
1. **Только тем, о ком говорили** — НИКОГДА не отправлять людям, которых НЕ упоминали в текущем разговоре
2. **Всегда спрашивай** — перед отправкой покажи: "Отправить [кому]: «[текст]»? (Да / Нет)"
3. **Никаких тестов** — НИКОГДА не отправлять слово "тест" или тестовые сообщения кому-либо
4. Если не уверен кому отправлять — спроси у Дмитрия
5. Все отправки логируются в `logs/tg-send.log`

### 🤖 Agent Teams — автоматическая оценка (перед задачей)
После скилл-чека — быстро оцени: **нужна ли команда агентов?**
Ответь 1 строкой: `🤖 Teams: ДА (причина) → создаю команду` или `🤖 Teams: НЕТ → выполню сам`

**Когда ДА** (создать команду):
- Задача затрагивает 2+ клиентов/ботов одновременно (обновить всех, проверить всех)
- Параллельный дебаг нескольких сервисов
- Рефакторинг 3+ независимых модулей
- Задача с явно разделяемыми подзадачами (фронт + бэк + тесты)
- Пользователь явно просит параллельную работу

**Когда НЕТ** (работать одному):
- Одна задача, один клиент, один файл
- Быстрый фикс, деплой, отправка сообщения
- Задача где шаги зависят друг от друга (последовательная)
- Задача занимает < 5 минут

**Правила команды:**
- 3-5 агентов максимум (токены дорогие)
- Каждый агент = свои файлы (не пересекаться!)
- Режим: tmux (через `--teammate-mode tmux`)
- Лид (главный агент) координирует и собирает результат

### 📖 Чтение логов и истории (ПОЛНОСТЬЮ, БЕЗ ЛИМИТОВ)
При восстановлении контекста (подтягивание вчера/ночью, recall, recovery):
1. **SESSION_LOG.md, NIGHT_LOG.md, HANDOFF.md** — читать БЕЗ ЛИМИТА строк (или grep по ключевым словам)
2. ⛔ **НЕ использовать limit=100 или limit=N** на исторических файлах — это скрывает критическую информацию
3. Если файл большой → `grep -E "дата|ключевое_слово" файл` вместо Read с лимитом
4. После чтения → `git log --all --oneline` / `git diff` если нужны детали в коде
5. Если сомневаешься что прочитал всё → спроси или проверь вторично

**Почему:**
- Лимит строк = слепое пятно. Задачи/решения могут быть в конце файла
- Критический контекст теряется → неверные ответы → переделать работу
- Исторические файлы СТАТИЧНЫ — их можно читать полностью без штрафа на токены
