# dm-sender — Отправка личных сообщений от имени Дмитрия

## Назначение

Отправить сообщение в личку (DM) любому контакту в Telegram от имени Дмитрия через его userbot (Telethon). Скилл решает типовые проблемы: выбор сессии, поиск контакта, правильное представление.

## Когда использовать

- «Отправь в личку Виктории»
- «Напиши от моего имени Марине»
- «Скажи ей/ему в ЛС»
- «Передай в телеграм»
- «Пошли сообщение [имя]»
- «Написать лично [кому]»
- «Send DM to [name]»
- Любая вариация «отправь от меня» + указание на личное сообщение (не группу, не канал)

НЕ использовать для:
- Отправки в HQ-группы (используй Bot API или `tg-send.py`)
- Отправки от имени бота клиента
- Массовых рассылок

### Фаза 2: Составление сообщения

Правила:

1. **Представление** — ВСЕГДА начинай с `Это агент Дмитрия 🤖` (если Дмитрий не сказал иначе)
2. **Тон** — от лица Дмитрия, дружелюбно, по делу
3. **Не выдумывай** — пиши только то, что Дмитрий попросил передать
4. **Без технического жаргона** — адаптируй под получателя

Если Дмитрий не дал точный текст — составь черновик и покажи ему:
```
Вот что отправлю:
---
[текст сообщения]
---
Отправляю?
```

Если Дмитрий дал содержание (не точный текст) — оформи и отправляй без подтверждения.

### Фаза 3: Отправка через Telethon

**Сессия:** ТОЛЬКО parser-сессия. НЕ основная (она залочена HQ-ботом).

```python
import asyncio
from telethon import TelegramClient

async def send_dm(recipient, message):
    # 1. Подключение — ТОЛЬКО parser-сессия
    session_path = '/root/Antigravity/.secrets/telegram_userbot_parser'
    api_id = 33869550
    api_hash = 'a5ee0191e1ebef66b0fc3ad9ec7ad388'

    client = TelegramClient(session_path, api_id, api_hash)
    await client.connect()

    # 2. Проверка авторизации
    if not await client.is_user_authorized():
        print('❌ Parser-сессия не авторизована')
        await client.disconnect()
        return False

    # 3. Поиск получателя
    try:
        entity = await client.get_entity(recipient)  # ID или @username
    except ValueError:
        # Контакт не в кэше — пробуем через username
        print(f'❌ Контакт {recipient} не найден в кэше сессии')
        await client.disconnect()
        return False

    # 4. Отправка
    await client.send_message(entity, message)
    print(f'✅ Отправлено: {entity.first_name}')

    await client.disconnect()
    return True

asyncio.run(send_dm(RECIPIENT, MESSAGE))
```

**Критические правила кода:**

| Правило | Почему |
|---------|--------|
| Путь сессии: `/root/Antigravity/.secrets/telegram_userbot_parser` | Основная сессия залочена HQ-ботом — DatabaseLockedError |
| API ID: `33869550` | Единый API ID Дмитрия для всех userbot |
| API Hash: `a5ee0191e1ebef66b0fc3ad9ec7ad388` | Парный к API ID |
| `await client.disconnect()` обязательно | Незакрытая сессия залочит файл для следующего вызова |
| `get_entity(int_id)` — не строку | Строковые ID не работают, нужен int |

### Фаза 4: Обработка ошибок

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Cannot find any entity` | Контакт не в кэше parser-сессии | Попробуй `@username` вместо ID. Если нет username — отправь через основную сессию (`/root/Antigravity/.secrets/telegram_userbot`) с проверкой что HQ-бот не запущен |
| `database is locked` | Другой процесс использует сессию | tg-send.py v2 решает автоматически (temp copy). Если вызываешь вручную — используй `tg-send.py` вместо прямого Telethon |
| `Session not authorized` | Сессия истекла | Сообщи Дмитрию: «Parser-сессия истекла, нужна реавторизация» |
| `FloodWaitError` | Слишком много запросов | Подожди указанное время. Сообщи Дмитрию |
| `UserPrivacyRestrictedError` | Получатель запретил сообщения | Сообщи Дмитрию |

### Фаза 5: Фиксация

После успешной отправки:
1. Записать в `SESSION_LOG.md`: «Отправлено в личку [кому] от имени Дмитрия»
2. НЕ логировать содержание сообщения (приватность)

## Основной способ: tg-send.py v2

Скрипт `scripts/tg-send.py` автоматически решает все проблемы с сессиями:

```bash
python3 scripts/tg-send.py <алиас> "текст"
python3 scripts/tg-send.py maxim "Привет!"
python3 scripts/tg-send.py victoria "[FILE:/tmp/doc.pdf] Подпись"
```

**Алиасы:** maxim, victoria, marina, yulia, argisht, nikita, yana, katya, ekaterina, radion (+ русские варианты)

**Автоматика:**
- Parser-сессия по умолчанию (не залочена HQ-ботом)
- Если DB locked → автоматически копирует сессию в temp и подключается
- Fallback на основную сессию если parser недоступна
- Temp-файлы удаляются после отправки

**НЕ НУЖНО** вручную решать проблемы с сессиями — скрипт делает это сам.

## Anti-Patterns

| Не делай | Почему | Делай вместо |
|----------|--------|-------------|
| Использовать основную сессию `telegram_userbot` | Залочена HQ-ботом → `DatabaseLockedError` | Использовать `telegram_userbot_parser` |
| Передавать TG ID как строку в `get_entity()` | Telethon парсит строки как username | Передавать `int(id)` |
| Отправлять без «агент Дмитрия» | Получатель не поймёт, кто пишет | Всегда представляться |
| Не закрывать `client.disconnect()` | Залочит сессию для следующего вызова | Всегда в finally-блоке |
| Отправлять без подтверждения Дмитрия (если текст неясен) | Может отправить не то | Показать черновик если нет чёткого текста |
| Логировать содержание сообщения | Приватность | Логировать только факт отправки |
