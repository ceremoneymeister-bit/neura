# channel-autopilot

## Purpose

Полностью автоматизированный контент-пайплайн для Telegram-каналов @neuro_tseh и @neural_cex. Парсит 18 AI-каналов, выбирает лучшие посты по скорингу, переписывает в стиле Дмитрия, генерирует терминальные карточки в стиле ЦЕХ, публикует по расписанию.

## When to Use

- "автопостинг", "контент для канала", "сгенерируй посты"
- "запусти автопилот", "@neuro_tseh", "@neural_cex"
- "AI-дайджест для канала", "посты из каналов"
- Ночные задачи по генерации контента

## Architecture

```
18 AI-каналов → [Collector] → Top-3 по score → [Rewriter/Claude] → Стиль ЦЕХ → [Image Gen/Pillow] → Terminal cards → [Poster/Telethon] → @neuro_tseh
                                                                                                                         ↓
                                                                                                                   [Report → HQ PDF]
```

## Core Workflow (Decision Gates)

### Режим 1: Автопилот (полный пайплайн)
Запуск: `python3 scripts/channel-autopilot.py [args]`

1. **Collect** — сбор постов из 17 каналов (наш ЦЕХ исключён) за `--hours` часов
   - **Gate:** Если 0 кандидатов после дедупликации → EXIT с "Нет свежих кандидатов"
   - **Gate:** Если collector вернул exit code ≠ 0 → ABORT
2. **Rewrite** — рерайт топ-N через Claude CLI (модель: sonnet, таймаут: 120с)
   - **Gate:** Если Claude вернул пустой ответ → пост пропускается (не abort)
   - Очистка артефактов: автоматически удаляются строки "🔍 Скилл:" и "🤖 Teams:" из вывода Claude
   - Fallback заголовка: если title пустой → берётся первое предложение[:60]
3. **Image** — генерация терминальных карточек (Pillow, 1080×1080)
   - Fallback: при ошибке Pillow → минимальная карточка с заголовком по центру
4. **Post** — публикация через Telethon (main session)
   - `time.sleep(5)` между постами (защита от session lock)
   - Если `--schedule-times` → планирование на указанное время (NSK)
   - Обновление dedup-лога после каждого успешного поста
5. **Report** — Markdown → PDF (notion-pdf) → HQ-группа (thread 962)

### Режим 2: Ручной рерайт
Дмитрий указывает конкретный пост или канал → Claude переписывает → публикация в @neural_cex. Использует ту же дедупликацию (`autopilot_posted.json`). Не требует запуска скриптов — работает через диалог с Claude.

### Режим 3: Collect-only (разведка)
`python3 scripts/channel-autopilot.py --collect-only` — только сбор и скоринг. Полезно для анализа: какие каналы дают лучший контент, какие темы в тренде.

## Lessons Learned (из реальных сессий)

### Telethon Session Lock (2026-03-21)
При быстрой последовательной отправке постов через Telethon возникает session lock.
**Решение:** `time.sleep(5)` между публикациями (было 2 — мало). Сессии чтения и записи разделены: parser для сбора, main для публикации.

### Лимит 3 поста/день — НЕ enforced в коде (уточнено 2026-03-24)
Скрипт НЕ ограничивает количество постов — `--max-posts 7` публикует все 7 (подтверждено 24.03). Лимит 3/день — рекомендация, а не код. Ограничивать на уровне вызова: передавать `--max-posts 3`. Для @neuro_tseh (тестовый, 2 подп) допустимо до 7.

### Пост без заголовка (2026-03-21)
Рерайт через Claude иногда возвращает пост без заголовка (пост #6, score 0.700).
**Решение:** после рерайта — обязательная проверка: `if not title: title = first_sentence[:60]`. Добавить fallback-генерацию заголовка.

### Ручной рерайт vs Автопилот (2026-03-21)
Помимо автопилота, посты переписываются вручную для @neural_cex (3 поста 21 марта). Два режима:
- **Автопилот** (`channel-autopilot.py`) — полный пайплайн, @neuro_tseh
- **Ручной** — Дмитрий указывает конкретный пост/канал, Claude переписывает, постим в @neural_cex
Оба режима используют одну дедупликацию (`autopilot_posted.json`).

### Night Orchestrator Integration (2026-03-21–22)
Скилл зарегистрирован как template task `tmpl_content_neuro_tseh` в ночном оркестраторе:
- Средняя длительность: ~372 секунд (~6 минут)
- 2/2 ночных запуска — success
- Всегда запускается как Agent #2 (параллельно с другими задачами)
- PDF-отчёт автоматически уходит в HQ-топик "🌙 Ночные сессии" (thread 962)

### Scoring работает (2026-03-22)

## Quality Standards

### Чеклист для каждого поста (перед публикацией)
- [ ] Есть заголовок (не пустой, 3-7 слов)
- [ ] Длина текста: 500–1500 символов
- [ ] HTML-формат корректный (`<b>`, `<i>`, ссылки)
- [ ] Максимум 1-2 эмодзи (не больше!)
- [ ] Нет хештегов
- [ ] Указан источник (название канала) в конце
- [ ] Нет артефактов Claude ("Скилл:", "Teams:", "Вот пост:")
- [ ] Терминальная карточка сгенерирована (1080×1080, ЦЕХ-стиль)

### Метрики пайплайна
| Метрика | Норма | Тревога |
|---------|-------|---------|
| Конверсия (собрано → опубликовано) | 8-15% | <5% (плохие каналы) или >25% (слабый скоринг) |
| Время полного цикла | 5-8 мин | >15 мин (таймауты Claude) |
| Рерайт: пустые ответы | 0-1 из 3 | 2+ из 3 (проблема с промптом) |
| Score топ-поста | >0.85 | <0.70 (мало вовлечённости в каналах) |

## Rewrite Style Guide (краткая выжимка)

Полный стиль-гайд встроен в `scripts/channel-autopilot-rewrite.py`. Ключевые правила:

- **Голос:** друг, делящийся находками. НЕ учитель и НЕ эксперт-лектор
- **Тон:** разговорный — «Что кайф:», «Юзайте», «если достигаешь лимитов — здесь можно добивать»
- **Эмодзи:** 1-2 максимум, без 🔥🚀💡 на каждую строку
- **Структура:** заголовок → суть 1-2 предложения → буллеты → CTA/ссылка
- **Буллеты:** простые дефисы, без эмодзи
- **Личный опыт:** «я использую», «я собрал», «помог мне»
- **Запрещено:** «Друзья!», «Всем привет!», хештеги, канцелярит, «Подведём итоги»

## Error Handling & Timeouts

| Компонент | Таймаут | При ошибке |
|-----------|---------|------------|
| Сбор (collect) | 300с на весь скрипт | ABORT — без кандидатов нечего постить |
| Рерайт (Claude CLI) | 120с на один пост | SKIP пост, продолжить остальные |
| Генерация изображений | — | Fallback → минимальная карточка (заголовок по центру) |
| Публикация (Telethon) | 60с на пост | Лог ошибки, продолжить следующий пост |
| HQ-уведомление | 10с | Лог ошибки, НЕ abort (некритично) |
| PDF-отчёт | 30с | Лог ошибки, отправить только текст в HQ |

## Troubleshooting

### «Нет подходящих кандидатов»
Все посты уже в `autopilot_posted.json` (дедупликация). Решения:
- Увеличить `--hours` (например, 48)
- Проверить размер dedup-лога (макс 500 записей, автоочистка)
- Добавить новые каналы в `AI_CHANNELS` в `channel-autopilot-collect.py`

### Telethon session lock / AuthKeyDuplicatedError
Две сессии одновременно пытаются использовать один файл. Решения:
- Убедиться что parser session (`_parser`) используется ТОЛЬКО для чтения
- Убедиться что main session используется ТОЛЬКО для публикации
- Увеличить `time.sleep()` между постами (текущий: 5с)

### Claude возвращает пустой ответ / таймаут
- Проверить что `claude` CLI доступен в PATH
- Проверить env: скрипт удаляет `CLAUDECODE` из env (иначе рекурсия)
- Модель: `sonnet` (быстрее и дешевле чем opus для рерайта)

### Карточки не генерируются
- Проверить шрифт: `ls /usr/share/fonts/truetype/dejavu/DejaVuSansMono.ttf`
- Проверить PIL: `python3 -c "from PIL import Image; print('OK')"`
- Fallback сработает автоматически — минимальная карточка всегда создаётся

### Cron не запускается
- Проверить: `crontab -l | grep autopilot`
- Логи: `tail -f logs/autopilot.log`
- Часовой пояс cron = UTC на сервере (NSK = UTC+7)

## Anti-Patterns

- НЕ постить больше 3 раз в день — спам убивает подписчиков
- НЕ копировать посты дословно — всегда рерайт в своём стиле
- НЕ использовать main session для чтения каналов — только parser
- НЕ постить в @neural_cex без предварительного тестирования на @neuro_tseh
- НЕ отправлять посты с `time.sleep < 5` — session lock при быстрой отправке
- НЕ публиковать пост без заголовка — всегда проверять наличие title после рерайта
- НЕ запускать >7 постов за одну ночную сессию — даже с расписанием, это перегружает дедупликацию
- НЕ удалять env `CLAUDECODE` вручную — скрипт rewrite.py сам чистит его (предотвращает рекурсию Claude Code)
- НЕ запускать автопилот без проверки `--dry-run` при изменении промпта или стиль-гайда
- НЕ полагаться на `--max-posts` default (3) как на жёсткий лимит — night orchestrator может передать любое число. Для @neural_cex ОБЯЗАТЕЛЕН хардкод-лимит на уровне скрипта (daily cap через dedup-лог)
- НЕ игнорировать срабатывание parser session fallback — это симптом конфликта сессий, расследовать причину

## Lessons Learned — дополнение (из анализа кода и логов 20-22 марта 2026)

### Parser Session Fallback (из кода orchestrator)
Оркестратор `channel-autopilot.py` содержит встроенный fallback в `post_to_channel()`:
если main session возвращает "database is locked" в stderr → автоматически переключается на parser session.
**Важно:** Это аварийный режим. Parser session НЕ предназначена для записи. Если fallback срабатывает регулярно — это симптом конфликта сессий (HQ-бот или другой скрипт держит main session). Диагностика: `fuser .secrets/telegram_userbot.session`.

### Schedule Tomorrow Logic (из кода orchestrator)
При использовании `--schedule-times`, если указанное время уже прошло сегодня (по NSK), пост автоматически планируется на **завтра**. Это защищает от ситуации "запустил в 03:00 NSK, указал 09:00 — пост ушёл бы в прошлое". Работает корректно, не трогать.

### Dual Trigger Risk: Cron vs Night Orchestrator (март 2026)
Существуют ДВА механизма запуска channel-autopilot:
1. **Cron** — `0 19 * * *` UTC (02:00 NSK) — прямой вызов `channel-autopilot.py`
2. **Night Orchestrator** — systemd timer на 20:00 UTC (03:00 NSK) — вызывает через шаблон `content_neuro_tseh`
Если оба активны — autopilot выполнится дважды за ночь. Дедупликация (`autopilot_posted.json`) предотвращает повторную публикацию, но тратятся токены Claude на холостой рерайт.
**Рекомендация:** использовать только ONE из двух. Текущий приоритет — Night Orchestrator (он гибче + генерирует HQ-отчёт).

### Шаблон @neural_cex — отдельная задача (март 2026)
В `task-templates.yaml` есть два шаблона:
- `content_neuro_tseh` → channel-autopilot (автоматизированный пайплайн, @neuro_tseh)
- `content_neural_cex` → brand-voice-clone (авторский контент, @neural_cex)
Это **не дублирование** — каналы имеют разные стратегические роли:
- `@neuro_tseh` (2 подп) = тестовый полигон, автоматизированный AI-дайджест
- `@neural_cex` (67 подп) = основной канал Дмитрия, авторский голос, ручной контроль
Автопилот НЕ ДОЛЖЕН постить в @neural_cex без явного `--channel @neural_cex` от Дмитрия.

### Лучшие источники для ручного рерайта (из SESSION_LOG 21-22 марта)
По данным 4 ручных рерайтов:
- **AI бля** — 2 поста (провокационные темы, высокий engagement: агент vs инструмент, армии агентов)
- **Философия Ханта** — 1 пост (интеллектуальный, "Воззрение")
- **ИИздец** — 1 пост (горячие тейки, "AI уже заменяет")
**Паттерн:** для ручного рерайта лучше работают каналы с сильной авторской позицией (AI бля, Философия Ханта), а не агрегаторы новостей.

### HQ-отчёт: два уровня (из кода)
Оркестратор отправляет в HQ **два** сообщения:
1. **Текстовое уведомление** → thread 962 (Ночные сессии) — краткий статус (✅/❌ по шагам, кол-во постов)
2. **PDF-документ** → thread 962 — полный отчёт с текстами постов (первые 300 символов каждого)
Если PDF-генерация падает → отправляется только текст (graceful degradation). Скрипт md2pdf.py из notion-pdf с таймаутом 30с.

### Таймаут 300с на subprocess (из кода)
Каждый sub-script (collect, rewrite, image) запускается через `subprocess.run(timeout=300)`. Если один скрипт зависает >5 минут — оркестратор получит TimeoutExpired exception, которое НЕ обрабатывается (crash). Потенциальное улучшение: обернуть в try/except для graceful abort.

### Dedup-лог: состояние и масштаб (2026-03-23)

### Subprocess TimeoutExpired — конкретный код-гэп (2026-03-23, из аудита кода)
В `channel-autopilot.py` вызовы `subprocess.run(timeout=300)` для collect, rewrite и image НЕ обёрнуты в `try/except subprocess.TimeoutExpired`. ### Отсутствие systemd-watchdog для cron (2026-03-22, из архитектурного аудита)
`channel-autopilot.py` при запуске через cron не имеет systemd-обёртки. Если скрипт крашнется → тихий fail без уведомления. **Рекомендация:** создать `channel-autopilot.service` + `channel-autopilot.timer` (аналог cron, но с логами и watchdog). ### Убийцы и бустеры охватов (по Тимочко)

Перед публикацией каждого поста — проверить по двум чеклистам.

#### 12 убийц охватов (что роняет)

- [ ] **Скучный контент без эмоций** — нет личного мнения, тейка, позиции
- [ ] **Длинные посты без структуры** — стена текста, нет заголовков/буллетов
- [ ] **Нерегулярный постинг** — паузы > 3 дней убивают алгоритмы
- [ ] **Только продажи** — > 30% продающего контента = отписки
- [ ] **Нет визуала** — пост без картинки/карточки теряет 40-60% охвата
- [ ] **Непонятный посыл** — новый подписчик не поймёт о чём канал за 5 секунд
- [ ] **Негатив и жалобы** — хейт, нытьё, «всё плохо» без решения
- [ ] **Копипаст без адаптации** — перепостнул чужой текст как есть
- [ ] **Нет interaction** — ни одного вопроса, опроса, призыва к действию
- [ ] **Слишком много хештегов** — > 2 хештега = визуальный шум (скилл использует 0)
- [ ] **Постинг в мёртвое время** — ночь, раннее утро (оптимум: 09:00, 13:00, 19:00 NSK)
- [ ] **Нет серийности** — каждый пост как отдельный остров, нет связи между публикациями
