# Check-in Templates — проактивные касания

## health-check — как работает бот

**Когда:** Плановая проверка (7-14 дней с последнего контакта). Клиент активно пользуется ботом.

**Принципы:**
- Коротко: 2-3 предложения максимум
- Забота, не навязчивость
- Открытый вопрос (не да/нет)
- Показать что мы следим за качеством

### Шаблон

```
Это агент Дмитрия 🤖

{name}, как работает {product/bot_name}? Всё стабильно?

Если что-то хочешь улучшить или добавить — напиши, разберёмся.
```

### Варианты (ротация, не использовать один и тот же дважды подряд):

**v1 — стандартный:**
```
Это агент Дмитрия 🤖

{name}, как работает бот? Всё стабильно?

Если что-то хочешь улучшить — напиши, разберёмся.
```

**v2 — с конкретикой:**
```
Это агент Дмитрия 🤖

{name}, проверил — твой бот работает стабильно, {конкретная метрика: "обработал X запросов за неделю" / "без сбоев последние 7 дней"}.

Есть что-то, что хотелось бы улучшить?
```

**v3 — с предложением:**
```
Это агент Дмитрия 🤖

{name}, у нас появилась новая возможность — {краткое описание}. Думаю, тебе может быть полезно.

Хочешь попробовать?
```

## feature-notification — новая фича подключена

**Когда:** Агент подключил новую возможность для конкретного клиента.

**Принципы:**
- Конкретика: что именно подключено
- Польза: зачем это клиенту (1-2 предложения)
- Как попробовать (1 предложение)
- Без технических деталей

### Шаблон

```
Это агент Дмитрия 🤖

{name}, подключил новую возможность: {feature_name}

{what_it_does — что делает, 1-2 предложения}
{how_to_try — как попробовать, 1 предложение}

Попробуй и скажи, как 👍
```

### Пример

```
Это агент Дмитрия 🤖

Виктория, подключил новую возможность: генерация PDF-документов.

Теперь бот может создавать красиво оформленные PDF — планы, отчёты, чек-листы. Просто попроси бота "сделай PDF с планом на неделю".

Попробуй и скажи, как 👍
```

## reactivation — давно не общались

**Когда:** 7+ дней без контакта с клиентом. Никаких задач/тикетов в работе.

**Принципы:**
- Максимально коротко (2-3 предложения)
- Не извиняться за молчание
- Привязать к чему-то конкретному из контекста клиента
- Лёгкий вопрос — просто ответить

### Шаблон

```
Это агент Дмитрия 🤖

{name}, давно не общались — как {relevant_topic}?

Если нужна помощь с {product} — я на связи.
```

### Варианты:

**v1 — минималистичный:**
```
Это агент Дмитрия 🤖

{name}, как дела с {product}? Всё работает?
```

**v2 — с контекстом:**
```
Это агент Дмитрия 🤖

{name}, последний раз общались по {тема} — как в итоге? Получилось?
```

**v3 — с ценностью:**
```
Это агент Дмитрия 🤖

{name}, за последнюю неделю добавили несколько улучшений в систему. Хочешь обновлю и твоего бота?
```

## milestone — достижение или метрика

**Когда:** Бот клиента достиг показателя, который стоит отметить.

**Принципы:**
- Конкретная цифра
- Что это значит для клиента
- Позитив — праздновать маленькие победы
- Коротко

### Шаблон

```
Это агент Дмитрия 🤖

{name}, маленький milestone: {metric — "твой бот обработал 500 запросов" / "3 месяца стабильной работы" / "10 документов в базе знаний"}

{what_it_means — 1 предложение}

Так держать 💪
```

### Пример

```
Это агент Дмитрия 🤖

Марина, маленький milestone: твои сотрудники задали боту уже 200+ вопросов за месяц.

Это значит, что бот реально экономит время команде. Если хочешь — могу подготовить отчёт по самым частым запросам.

Так держать 💪
```
