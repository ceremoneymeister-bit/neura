# Business Templates — деловая переписка

## delivery — отправка результата

**Когда:** Дмитрий (или агент) выполнил задачу → нужно передать результат клиенту.

**Принципы:**
- Результат сразу, без предисловий
- Структура: списки, emoji-маркеры для визуального сканирования
- Если есть ссылки/доступы — выделить отдельными строками
- Чёткий next step или вопрос

### Шаблон

```
Это агент Дмитрия 🤖

{context — что сделано, 1 предложение}

{result — структурированный:
— пункт 1
— пункт 2
— ссылка/доступ если есть}

{next_step — "Проверь и скажи, всё ок?" / "Попробуй и напиши, как"}
```

### Пример

```
Это агент Дмитрия 🤖

Починил проблему с поиском в твоём боте — теперь находит релевантные документы.

Что исправлено:
— Убраны нерелевантные результаты из индекса
— Ускорен поиск (было 3 сек → стало 0.5 сек)

Попробуй задать боту вопрос по своей базе знаний и скажи, всё ок?
```

## status-update — прогресс по задаче

**Когда:** Задача в работе, нужно сообщить прогресс, не дожидаясь завершения.

**Принципы:**
- Что сделано + что это даёт клиенту
- Что дальше (если есть)
- НЕ перечислять технические детали
- Коротко — 2-5 предложений

### Шаблон

```
Это агент Дмитрия 🤖

{what_done} — {impact_for_client, 1 clause}

{what_next — "Продолжаем, следующий этап: X" / "Завершу к [когда]"}
```

### Пример

```
Это агент Дмитрия 🤖

Настроил автоматическую генерацию контента — теперь бот может писать посты для твоего канала.

Следующий этап: подключение расписания публикаций. Завершу сегодня к вечеру.
```

## follow-up — после встречи или звонка

**Когда:** Была встреча/звонок 1-2 дня назад, нужно зафиксировать договорённости.

**Принципы:**
- Персональное приветствие (не "Это агент", а от имени Дмитрия)
- Ключевые договорённости — список
- Конкретный next step с дедлайном
- Тон теплее обычного — после личного контакта

### Шаблон

```
{name}, привет!

По итогам {нашего звонка / встречи}:

{agreements:
1. ...
2. ...
3. ...}

{next_step — "С моей стороны сделаю X к [дата]. От тебя жду Y до [дата]"}
```

### Пример

```
Марина, привет!

По итогам нашего созвона:

1. Подключаю новый скилл для работы с клиентской базой
2. Ты отправляешь мне шаблоны документов до пятницы
3. Презентацию для сотрудников готовлю к понедельнику

С моей стороны пункты 1 и 3 будут готовы к понедельнику. Жду шаблоны от тебя 👍
```

## request — просьба / запрос информации

**Когда:** Нужно что-то от получателя: информация, доступ, решение, обратная связь.

**Принципы:**
- Сначала ЗАЧЕМ нужно (1 предложение)
- Потом ЧТО конкретно нужно
- Дедлайн если есть
- Максимально просто ответить (да/нет, выбор из вариантов, одно слово)

### Шаблон

```
Это агент Дмитрия 🤖

{why — зачем это нужно, 1 предложение}

{what — что конкретно нужно:
— пункт 1
— пункт 2}

{deadline — "Если можно до [дата] — было бы идеально" / без дедлайна если не срочно}
```

### Пример

```
Это агент Дмитрия 🤖

Для подключения твоего бота к Google Drive нужны доступы.

Пришли, пожалуйста:
— Email аккаунта Google
— Папку на Drive, где хранишь документы для бота

Если можно сегодня-завтра — подключу сразу.
```
