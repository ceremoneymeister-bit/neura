# Anti-Patterns — чего НЕ делать в сообщениях

Этот файл пополняется автоматически из `data/corrections.jsonl`. Каждый паттерн = реальная ошибка, которую Дмитрий скорректировал.

---

## Универсальные антипаттерны

### Пустые вступления
- "Привет! Как дела?" → без контекста = пустое сообщение
- "Хотел написать тебе..." → фраза-паразит, ноль информации
- "Надеюсь, у тебя всё хорошо!" → filler, не уважает время
- **Вместо:** сразу контекст + ценность

### Избыточные восклицания
- "Привет!!!" → один восклицательный знак максимум
- "Это так круто!!!" → энтузиазм через содержание, не через пунктуацию
- **Правило:** максимум 1 восклицательный знак на сообщение (в CTA допустимо)

### Шаблонность
- Дословное копирование шаблона без адаптации
- Одинаковые формулировки разным людям в один день
- **Вместо:** адаптировать под контекст, упоминать конкретику (имя бота, последнюю задачу)

### Длина не по типу
- Check-in на 10+ предложений → получатель не дочитает
- Sales intro > 8 предложений → потеря внимания
- Delivery без структуры → сложно извлечь информацию
- **Вместо:** ориентиры из CLEAR-scoring.md, секция R

### Размытый CTA
- "Ну вот, как-то так..." → получатель не знает что ответить
- "Если что — пиши" (без конкретики) → слабый CTA
- **Вместо:** конкретный вопрос или действие: "Проверь и скажи, работает?"

### Навязчивость
- Больше 1 check-in за 7 дней одному клиенту
- Повторный sales-питч без нового контекста
- Follow-up на follow-up без ответа
- **Правило:** после 2 сообщений без ответа — СТОП. Подождать ответа или явного указания Дмитрия

### Sales клиенту
- Питчить Neura клиенту, который уже платит
- Предлагать тест-драйв тому, кто уже пользуется
- **Правило:** sales-шаблоны ТОЛЬКО для leads и prospects, НИКОГДА для текущих клиентов
