# Smart Outreach — проактивная коммуникация в Telegram

## Назначение

Мозг личной коммуникации. `dm-sender` = руки (отправляет через Telethon), `smart-outreach` = мозг (решает **что**, **как** и **когда** писать). Каждое сообщение проходит оценку качества CLEAR, учитывается в трекере, улучшает шаблоны.

## Когда использовать

**Прямые триггеры (Дмитрий просит):**
- "напиши [кому]", "отправь [кому]", "сообщение для [кого]"
- "check-in [кому]", "follow-up [кому]"
- "как дела у [клиент]?", "давно не писали [кому]"
- "предложи [кому] тест-драйв", "питч для [кого]"
- "outreach [кому]", "продай [кому]"

**Внутренние триггеры (скилл предлагает сам):**
- После `release-notes` → предложить `feature-notification` клиентам
- После `systematic-debugging` → предложить `delivery` (фикс) конкретному клиенту
- Тишина 7+ дней с клиентом → предложить `reactivation`
- Встреча 2+ дня назад без follow-up → предложить `follow-up`
- Лид 3+ дня без ответа → предложить `warm-follow-up`

**НЕ использовать для:**
- Постинг в каналы → `channel-autopilot`
- Рассылка в HQ-группы → `release-notes`
- Массовый спам (никогда)

```

│

│   ├── Прогресс по задаче → templates/business.md#status-update
│   └── После встречи/звонка → templates/business.md#follow-up
│
├── Спросить / попросить
│   └── Нужна информация / доступ / решение → templates/business.md#request
│
├── Узнать как дела / напомнить о себе
│   ├── Проверка работы бота → templates/check-in.md#health-check
│   ├── Новая фича подключена → templates/check-in.md#feature-notification
│   ├── Давно не общались → templates/check-in.md#reactivation

└── Продать / пригласить / закрыть
    ├── ⚡ Определить hunter_stage контакта (1-5) → см. «Лестница Ханта»
```

## Core Workflow (4 фазы)

### Фаза 1: Контекст

Перед генерацией сообщения — собрать:

1. **Кто получатель?** → Проверить `data/recipient-profiles.json`
   - Есть профиль → загрузить предпочтения (формальность, длина, антипаттерны)
   - Нет профиля → использовать дефолты, спросить у Дмитрия если нужно
2. **Какой тип сообщения?** → Decision Tree выше
3. **Какой контекст?** → Что произошло, зачем пишем (из разговора с Дмитрием или из триггера)
4. **Последнее взаимодействие?** → Проверить `data/tracker.jsonl` по recipient
5. **Стадия лестницы Ханта?** → Проверить `hunter_stage` в профиле (1-5). Если нет — определить по истории:
   - Нет истории → стадия 1
   - Был диалог без интереса к продукту → стадия 2
   - Проявил интерес → стадия 3
   - Были вопросы/возражения → стадия 4
   - Готов к покупке → стадия 5
6. **Follow-up Timeline?** → Проверить время с последнего касания → определить следующее действие по таймлайну

### Фаза 2: Черновик

1. Загрузить соответствующий шаблон из `templates/`
2. Заполнить переменные контекстом из Фазы 1
3. Адаптировать тон под recipient-profile
4. Если sales → подгрузить `marketing-psychology` для усиления
5. **CTA Stress Check:** проверить что CTA соответствует `hunter_stage` контакта (см. таблицу CTA Stress)
6. **Product Matrix:** если есть оффер → проверить что уровень продукта соответствует теплоте
7. **Категория прогрева:** если check-in → выбрать конкретную категорию (1-11) вместо "как дела?"

### Фаза 3: CLEAR-проверка

Оценить черновик по 5 критериям (прочитай `framework/clear-scoring.md`):

```
CLEAR: C=? L=? E=? A=? R=? → Total: ?/25
```

- **≥ 20/25** → показать Дмитрию
- **15-19** → переработать автоматически (обычно проблема в E или R)
- **< 15** → переписать с нуля

### Фаза 4: Подтверждение и отправка

Показать Дмитрию в формате:

```
📨 Smart Outreach

Кому: {имя} ({тип отношений})
Тип: {шаблон}
Хант: стадия {N}/5 → {название стадии}
Прогрев: #{категория} — {название категории}
CTA: {уровень} (соответствует стадии: ✅/⚠️)
CLEAR: {score}/25

---
{текст сообщения}
---

Отправить? (Да / Редактировать / Пропустить)
```

**После "Да":**
1. Отправить через `dm-sender` (Telethon parser-сессия)
2. Записать в `data/tracker.jsonl`
3. Вывести подтверждение

**После "Редактировать":**
1. Дмитрий даёт правки → внести
2. Записать коррекцию в `data/corrections.jsonl`
3. Повторить Фазу 4

**После "Пропустить":**
1. Записать в tracker как `skipped`
2. Если 3+ пропусков одного типа → снизить приоритет этого триггера

## Стратегические фреймворки

### Лестница Ханта (Hunter's Ladder) — стадия DM

Каждое DM-сообщение привязано к стадии лестницы. **Правило:** не перескакивать. Каждое следующее сообщение — на 1 ступень выше предыдущего разговора.

| Стадия | Цель | Шаблон сообщения | CTA |
|--------|------|------------------|-----|
| 1. Ниша | Поделиться инсайтом отрасли | "Видел, что [их ниша] сейчас [тренд]. Ты уже используешь AI?" | Вопрос |
| 2. Эксперт | Показать экспертизу | "Кстати, сделал [кейс] для похожего бизнеса. Интересно?" | Показать кейс |
| 3. Продукт | Мягкое знакомство с продуктом | "У нас есть штука, которая решает [их боль]. Показать?" | Предложить демо |
| 4. Возражения | Снять сомнения | "Кстати, [имя] тоже так думал, пока не попробовал. Вот что получилось." | Социальное доказательство |
| 5. Продажа | Прямой оффер | "Тест-драйв 7 000₽/нед. Настраиваем за день. Попробуешь?" | Купить / записаться |

**Правила применения:**
- В `recipient-profiles.json` хранить поле `hunter_stage` (1-5) для каждого контакта
- При генерации сообщения — проверить текущую стадию, писать ТОЛЬКО на текущей или +1
- Если контакт холодный и нет истории → стадия 1
- Если был позитивный ответ на стадии N → следующее сообщение = стадия N+1
- Если `no_response_48h` → остаться на текущей стадии, сменить подход

### CTA Stress — давление призыва к действию

Интенсивность CTA должна соответствовать теплоте контакта. Несоответствие убивает отношения.

| Теплота контакта | Допустимые CTA | Запрещённые CTA |
|------------------|---------------|-----------------|
| Холодный (стадии 1-2) | Вопрос, поделиться ценностью, спросить мнение | "Купи", "Запишись", "Оплати", ссылка на оплату |
| Тёплый (стадия 3) | Предложить показать, пригласить на демо | Прямая продажа, дедлайны |
| Горячий (стадии 4-5) | Снять возражение, оффер, дедлайн | — |

**⛔ Абсолютный запрет:** первое сообщение холодному контакту с CTA "Купи" / "Запишись" / "Оплати". Это убивает отношения навсегда.

**В CLEAR-проверке:** добавить проверку CTA Stress как дополнительный фильтр:
- Если `hunter_stage` контакта = 1-2, а CTA = продажа → **заблокировать**, переписать на ценность
- Если `hunter_stage` = 3-4, а CTA = вопрос → допустимо, но предупредить что можно усилить

### 11 категорий прогрева — замена "как дела?"

Вместо пустых check-in использовать ценностные категории. Каждое касание несёт пользу получателю.

| # | Категория | Пример DM | Когда использовать |
|---|-----------|-----------|-------------------|
| 1 | Личное | Поздравить с ДР, событием | Известна дата / событие |
| 2 | Лайфстайл | "Видел твой пост про [X] — круто!" | После публикации получателя |
| 3 | Инсайт рынка | "В [нише] сейчас [тренд]. Заметил?" | Есть свежая новость / тренд |
| 4 | Экспертность | "Что думаешь про [тренд в их нише]?" | Хочешь вовлечь в диалог |
| 5 | Инфоповод | "[Событие] в индустрии. Как это на тебе отразится?" | Есть инфоповод |
| 6 | Клиенты | "Сделали [результат] для [похожий бизнес]. Интересно?" | Есть свежий кейс |
| 7 | Процесс | "Сейчас делаем [процесс]. Расскажу когда будет готово" | Строишь в открытую |
| 8 | Продукт | "Добавили [фича]. Это для [их боль]." | Вышла релевантная фича |
| 9 | Полезность | "Нашёл [инструмент/ресурс] — тебе пригодится." | Есть полезная ссылка / гайд |
| 10 | Мероприятие | "Делаем [бесплатное событие]. Присоединишься?" | Есть вебинар / эфир / встреча |
| 11 | Социальное доказательство | "[Имя] попробовал и получил [результат]." | Есть отзыв / кейс |

**Правило выбора:** при генерации check-in → выбрать категорию на основе доступного контекста. Если нет контекста ни для одной категории → **не писать** (лучше молчание, чем пустое "как дела?").

### Product Matrix — оффер под теплоту

Уровень предложения должен соответствовать уровню отношений.

| Теплота | Продукт | Пример оффера | Цель |
|---------|---------|---------------|------|
| Холодный | Lead magnet | "Кстати, сделал гайд по [тема]. Скинуть?" | Дать ценность, начать диалог |
| Тёплый | Tripwire | "Хочешь попробовать? Тест-драйв 7k/нед." | Первая оплата, снять барьер |
| Горячий | Main product | "Давай обсудим полную настройку." | Основная продажа |
| Клиент | Maximizer | "Добавили новый скилл — хочешь попробовать?" | Допродажа, удержание |

**Связь с лестницей Ханта:**
- Холодный (стадии 1-2) → Lead magnet
- Тёплый (стадия 3) → Tripwire
- Горячий (стадии 4-5) → Main product
- Клиент (после покупки) → Maximizer

### Follow-up Timeline — ритм касаний

Таймлайн после первого контакта или демо. Не спамить, но и не забывать.

| Время | Действие | Шаблон |
|-------|----------|--------|
| Сразу | Ценностное сообщение или lead magnet | Категория прогрева #6-9 |
| 24ч | "Посмотрел? Есть вопросы?" | `sales/warm-follow-up` |
| 3 дня | Кейс или персонализированный анализ | Категория прогрева #6, #11 |
| 7 дней | Ограниченный оффер или дедлайн | `sales/closing` (если стадия ≥ 4) |
| 14+ дней | **Стоп DM.** Перевести в контентную воронку | Добавить в аудиторию канала |

**Правило 14 дней:** если после 4 касаний нет ответа → прекратить DM, добавить в подписчики канала. Возобновить только при появлении нового инфоповода (категория #5) или достижении получателя (категория #2).

**Источники:** Пыриков (лестница Ханта, категории прогрева), НЕЧ20 (CTA stress, Product Matrix)

## Голос по умолчанию

**От агента (🤖):**
- Вступление: `Это агент Дмитрия 🤖` (для клиентов)
- Вступление: `Это агент Дмитрия Ростовцева 🤖` (для новых контактов)
- Тон: профессиональный, структурированный, тёплый, но не панибратский
- Длина: минимальная для передачи смысла
- Без: восклицательных знаков подряд (!!!), слова "надеюсь", фразы "как дела?"

**Адаптация по получателю:**
- `formality: 1-3` → можно без "Это агент Дмитрия", более casual
- `formality: 4-7` → стандартный формат
- `formality: 8-10` → "Добрый день" вместо имени, вы-форма

## Самообучение (исполняемое)

Движок: `engine.py` — Python-скрипт с CLI для всех операций обучения.

### Исполняемые команды

| Когда | Команда | Что делает |
|-------|---------|-----------|
| После отправки | `python3 engine.py log <recipient> <type> "C=5,L=4,E=5,A=4,R=5"` | Записать в tracker + обновить last_contact |
| Получен ответ | `python3 engine.py outcome <msg_id> responded_positive` | Обновить outcome + пересчитать effective_score |
| Коррекция | Через `engine.log_correction()` в Python | Записать + авто-промоушен если severity=high |
| Начало сессии | `python3 engine.py triggers` | Проверить все проактивные триггеры |
| По запросу | `python3 engine.py report` | Еженедельная статистика |
| По запросу | `python3 engine.py promote` | Перенести повторяющиеся коррекции в anti-patterns.md |
| По запросу | `python3 engine.py update-profiles` | Пересчитать профили из tracker данных |
| Перед отправкой | `python3 engine.py time-check [tz_offset]` | Проверить безопасное время (08:00–21:00) |

### Outcomes и скоринг

| Outcome | Score | Когда |
|---------|-------|-------|
| `responded_fast` | 9 | Ответ < 2 часов |
| `responded_positive` | 7 | Позитивный ответ |
| `responded_neutral` | 5 | Нейтральный ответ |
| `responded_negative` | 3 | Негативный ответ |
| `no_response_48h` | 2 | Нет ответа 48ч |
| `correction_needed` | 1 | Дмитрий переписал |
| `skipped` | 0 | Дмитрий пропустил |

### Цикл обучения

```
Отправка → engine.py log → tracker.jsonl
    ↓
48ч → engine.py outcome → effective_score
    ↓
Агрегация → engine.py update-profiles → recipient-profiles.json
    ↓
Коррекция → engine.py promote → anti-patterns.md → шаблоны улучшаются
```

### Автообновление (ОБЯЗАТЕЛЬНЫЕ шаги после каждого использования)

1. **После отправки:** `engine.py log` — записать CLEAR + recipient + type
2. **После коррекции:** сохранить original + corrected + pattern в corrections.jsonl
3. **При следующем контакте:** если узнал outcome — `engine.py outcome`
4. **Если 3+ коррекций за неделю:** `engine.py promote` — перенести в anti-patterns
5. **Если профиль изменился:** `engine.py update-profiles` — пересчитать статистику

### Еженедельный обзор

Триггер: "как работает outreach?", "статистика сообщений", "отчёт по outreach"

Запустить `python3 engine.py report` → получить:
- Всего сообщений за неделю
- Avg CLEAR score
- Response rate по типам
- Лучший/худший шаблон
- Рекомендации: что переписать, что масштабировать

## Anti-Patterns

| Не делай | Почему | Делай вместо |
|----------|--------|-------------|
| Отправлять без CLEAR-проверки | Качество деградирует | Всегда прогонять через CLEAR |
| Писать "Привет! Как дела?" без контекста | Пустое сообщение, не уважает время | Сразу к делу: контекст + ценность |
| Копировать шаблон дословно | Получатель заметит шаблонность | Адаптировать под контекст и профиль |
| Слать check-in чаще 1 раза в неделю | Навязчивость, клиент отключится | Max 1 check-in в 7 дней на клиента |
| Игнорировать no_response | Потеря данных для обучения | Фиксировать outcome через 48ч |
| Sales-питч клиенту который уже платит | Неуместно, раздражает | Sales только для лидов и prospects |
| Перескакивать стадии Ханта (1→5) | Контакт не готов, откажет | Каждое сообщение = +1 стадия максимум |
| Первое DM холодному = "Купи/Запишись" | Убивает отношения навсегда | Первое DM = ценность (стадия 1-2) |
| Check-in без категории прогрева | Пустое "как дела?" не уважает время | Выбрать категорию 1-11, нести ценность |
| 5+ DM без ответа подряд | Спам, блокировка | Стоп после 4 касаний, перевод в контент-воронку |
| Tripwire/Main product холодному контакту | Слишком рано, испугает | Lead magnet → Tripwire → Main (по Product Matrix) |

## Портирование в капсулы

Скилл существует в двух версиях:
1. **Antigravity** (этот файл) — полная версия с Telethon, sales, 6 получателей
2. **Capsule** (`neura-capsule/skills/smart-outreach/`) — лёгкая версия с Bot API

### Что портируется
- CLEAR-фреймворк (целиком)
- Шаблоны business + check-in (с `{{переменными}}`)
- engine.py (без изменений, пути относительные)
- Anti-patterns (целиком)

### Что НЕ портируется
- Sales-шаблоны (капсула уже продана)
- Telethon-отправка (капсулы используют Bot API)
- Hardcoded recipient-profiles (у каждой капсулы свой owner)
- Cold outreach (Bot API требует инициации от получателя)

### Как подключить к новой капсуле
1. Скопировать `neura-capsule/skills/smart-outreach/` в капсулу
2. Добавить `outreach` секцию в `config.json` капсулы
3. В `proactive.py` добавить `outreach_check` тип
4. Создать `data/outreach/` в капсуле для tracker + profiles
5. В CLAUDE.md капсулы добавить скилл в таблицу маршрутизации
