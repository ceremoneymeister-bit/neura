# synapse-curriculum

## Назначение

Полная учебная база школы {{SCHOOL_NAME}} — AI-мышление для детей 9–14 лет, Новосибирск. Содержит 4-месячную программу (16 занятий), шаблоны уроков, мастеркласса, матрицу инструментов, систему оценки и каталог проектов. Преподаватель — Анастасия Канева — может взять любой план урока и вести по нему без дополнительной подготовки.

## Когда использовать

- Разработка или корректировка учебной программы {{SCHOOL_NAME}}
- Создание плана конкретного урока (90 мин)
- Подготовка мастеркласса (4 часа, бесплатный для привлечения)
- Подбор AI-инструментов для конкретного занятия
- Оценка прогресса ученика
- Генерация идей проектов под возраст/интересы ребёнка
- Онбординг нового преподавателя (передать эту папку = передать всю методологию)

## Каноничный источник

**Перед любой работой прочитай:** `projects/Synapse/SYNAPSE_CANONICAL.md`

Все параметры (цены, формат, расписание, команда, подписки) берутся ТОЛЬКО оттуда. Этот скилл содержит МЕТОДОЛОГИЮ, а не бизнес-параметры.

## Ключевые принципы программы

1. **AI-мышление, НЕ программирование** — учим думать вместе с AI, а не кодить
2. **Проектный подход** — каждый урок завершается конкретным артефактом
3. **Две возрастные группы** — младшая (9–11): больше визуала, готовые промпты, пошаговое ведение; старшая (12–14): больше автономии, сложнее промпты, системное мышление
4. **Спиральная прогрессия** — навыки углубляются от месяца к месяцу (знакомство → создание → системы → запуск)
5. **Демо-день** — финальное занятие = презентация родителям, сертификаты

## Рабочий процесс

### Шаг 1: Определить задачу

| Задача | Что читать |
|--------|-----------|
| Подготовить конкретный урок | `references/module-catalog.md` → найти занятие → `references/lesson-template.md` → заполнить |
| Подготовить мастеркласс | `references/masterclass-template.md` |
| Подобрать инструменты | `references/tool-matrix.md` |
| Оценить ученика | `references/assessment-rubric.md` |
| Придумать проект | `references/project-catalog.md` → фильтр по возрасту и интересам |
| Адаптировать под возраст | Каждый файл содержит секции «Младшая» и «Старшая» |

### Шаг 2: Подготовка урока (типовой сценарий)

1. Открыть `module-catalog.md` → найти нужное занятие по номеру
2. Скопировать структуру из `lesson-template.md`
3. Заполнить блоки: разминка → навык → проект → шеринг → wrap-up
4. Проверить инструменты в `tool-matrix.md` (доступность из РФ, возрастная группа)
5. При необходимости взять проект из `project-catalog.md`
6. Добавить критерии оценки из `assessment-rubric.md`

### Шаг 3: Проверка перед уроком

- [ ] Все AI-инструменты работают и доступны из РФ
- [ ] Подготовлены аккаунты / подписки для учеников
- [ ] Есть готовые промпты-примеры (особенно для младшей группы)
- [ ] Артефакт урока чётко определён (что ученик уносит домой)
- [ ] Временные блоки реалистичны (90 мин = жёстко)

## Структура 90-минутного занятия

```
0:00–0:10  Разминка (check-in, AI-челлендж)
0:10–0:25  Навык (новая техника + мини-практика)
0:25–1:05  Проектная работа (40 мин — основной блок)
1:05–1:20  Шеринг (каждый показывает результат)
1:20–1:30  Wrap-up (итоги, превью следующего урока)
```

## Антипаттерны

1. **Учить кодить вместо думать** — {{SCHOOL_NAME}} НЕ школа программирования. Если занятие превращается в написание кода — остановись и вернись к AI-мышлению
2. **Одинаковый подход для 9 и 14 лет** — ВСЕГДА адаптировать: младшие = готовые промпты + пошагово; старшие = свобода + усложнение
3. **Инструмент без проекта** — каждый урок ОБЯЗАН заканчиваться конкретным артефактом. "Мы попробовали ChatGPT" — НЕ результат. "Мы создали комикс из 4 кадров" — результат
4. **Забыть про доступность из РФ** — перед планированием проверь `tool-matrix.md`. Midjourney через Discord может быть нестабильным
5. **Перегрузить занятие** — 90 минут = жёсткий лимит. Лучше одна техника глубоко, чем три поверхностно
6. **Пропустить шеринг** — презентация результатов = критическая часть. Без неё теряется мотивация и обратная связь
