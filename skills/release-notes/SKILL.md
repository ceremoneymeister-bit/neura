# Release Notes — рассылка обновлений в HQ-группы клиентов

## Назначение

После завершения системной работы (инфраструктура, боты, поиск, новые фичи, баг-фиксы) — автоматически отправить краткое, читабельное обновление в топик «Обновления» каждой клиентской HQ-группы. Формат — как Apple iOS release notes: что нового, что исправлено, что улучшено.

## Когда использовать

Триггеры — завершена работа, которая затрагивает:
- Серверную инфраструктуру (RAM, диск, сервисы, systemd)
- Векторный поиск (vsearch, индексация, модели)
- Ботов клиентов (новые фичи, баг-фиксы, оптимизации)
- Сайты и деплой (новые страницы, исправления анимаций, SEO)
- Новые инструменты или скиллы, влияющие на работу агентов
- Безопасность (патчи, обновления зависимостей)

НЕ триггерить на:
- Мелкие правки контента (посты, тексты)
- Внутренние рефакторинги без видимого эффекта
- Незавершённую работу (WIP)

## Workflow

### Фаза 1: Составить сообщение

Формат — Apple-style release notes. HTML-разметка (Telegram parse_mode=HTML).

```
📦 <b>Обновление системы</b>

<b>🆕 Что нового:</b>
• Описание новой фичи или возможности — без технического жаргона

<b>🔧 Исправлено:</b>
• Какая проблема была и как проявлялась (БЕЗ конкретных примеров сайтов/клиентов)
• Например: «Поиск по базе знаний мог возвращать нерелевантные результаты» — НЕ «поиск находил японский текст из google-cloud-sdk»

<b>⚡ Улучшения:</b>
• Метрика до → после (если уместно)
• Что стало быстрее/стабильнее/точнее

<i>— Команда разработки</i>
```

Правила составления:
1. **Язык клиента** — простой, понятный. Без «чанков», «systemd», «OOM kill»
2. **Без привязки к конкретным клиентам** — одно сообщение для всех
3. **Без конкретных примеров ошибок** — описывать класс проблемы, не инстанс
4. **Позитивный тон** — фокус на улучшениях, не на проблемах
5. **Краткость** — 5-10 пунктов максимум, каждый в 1 строку
6. **Секции опциональны** — если нечего писать в «Исправлено», не включай пустую секцию

### Фаза 2: Отправить

Вызвать скрипт рассылки:

```bash
python3 /root/Antigravity/.agent/skills/release-notes/scripts/send-release-notes.py "ТЕКСТ_СООБЩЕНИЯ"
```

Скрипт автоматически:
- Отправит в топик «Обновления» всех настроенных HQ-групп
- Создаст топик, если его ещё нет
- Пропустит группы с ошибками (graceful degradation)
- Выведет отчёт: кому отправлено, кому нет

### Фаза 3: Логирование

После отправки добавить строку в SESSION_LOG.md:
```
YYYY-MM-DD HH:MM — Release notes отправлены: [список групп]
```

## Anti-Patterns

| Не делай | Почему | Делай вместо |
|----------|--------|-------------|
| Отправлять техническую информацию (порты, токены, пути файлов) | Клиенты не понимают и пугаются | Переводить на человеческий язык |
| Упоминать конкретного клиента в общем обновлении | Другие клиенты видят то же сообщение | Писать обобщённо: «в работе ботов», «в поиске по базе знаний» |
| Слать обновление на каждый мелкий коммит | Спам — клиенты отключат уведомления | Группировать: одно обновление на сессию/день |
| Трогать бота Максима на сервере 45.11.93.179 | ⛔ Жёсткое правило — чужой сервер | Отправлять только через Bot API (не требует доступа к серверу) |

## Конфигурация

Файл: `.agent/skills/release-notes/config.json`

Содержит список HQ-групп, их bot_token и topic_id для «Обновления». Если topic_id = null — скрипт создаст топик автоматически. Если enabled = false — группа пропускается.
