# Brand Voice Clone

Система извлечения и применения авторского стиля из Telegram-каналов.

## Триггеры

Активировать при: "в стиле [канала/автора]", "voice clone", "brand voice", "напиши как", "стиль канала", "извлечь стиль", "голос канала", "профиль стиля", "клонировать голос".

## Два режима работы

### Режим 1: Извлечение (создание нового профиля)

Когда пользователь хочет проанализировать канал и создать профиль.

**Шаги:**

1. **Парсинг канала**
   ```bash
   python3 scripts/tg-parse-channel.py "Название канала" --all
   # или
   python3 scripts/tg-parse-channel.py --channel-id -100XXXXXXXXXX --all
   ```
   Результат: JSON в `.secrets/channel_dumps/{slug}_{date}.json`

2. **Количественный + качественный анализ**
   ```bash
   python3 scripts/tg-style-analyzer.py .secrets/channel_dumps/{slug}_{date}.json
   ```
   Результат: `{slug}_{date}_analysis.md` — метрики + Claude-анализ

3. **Лингвистическая реконструкция** (техника из X1000 #33)
   - Прочитать анализ
   - Выделить **функциональные маркеры**: ритм фраз, длина предложений, паттерны пауз (—, ..., ¶)
   - Определить **ядро стиля**: базовый тон + мета-стиль (ирония, провокация, забота, экспертиза)
   - Зафиксировать **лексический отпечаток**: слова-маркеры, частотные обороты, запрещённые конструкции
   - Выбрать 10-15 лучших reference posts
   - Написать профиль в `profiles/{slug}.md` по шаблону ниже

### Режим 2: Применение (написание в стиле профиля)

Когда пользователь хочет написать контент в стиле существующего профиля.

**Шаги:**

1. Найти профиль в `profiles/` или `memory/`
2. Прочитать ВЕСЬ профиль включая reference posts
3. Определить платформу → прочитать `references/platform-adaptation.md`
4. Написать контент, строго следуя:
   - Do's из профиля
   - Тону и энергии
   - Структурным паттернам
   - Лексическим маркерам
5. **Self-check:** сравнить с reference posts — узнаваем ли стиль?

## Интеграция со скиллами

### Приоритет загрузки
Brand Voice Clone загружается **ПЕРЕД** другими контент-скиллами:
1. `brand-voice-clone` → определяет голос
2. `social-content` / `copywriting` → определяет структуру и платформу
3. Профиль голоса **переопределяет** дефолтный тон из social-content

### Взаимодействие
- Hook-формулы из `social-content` адаптируются под лексику профиля
- Длина и структура из `copywriting` корректируются по метрикам профиля
- Стоп-слова и табу из профиля имеют абсолютный приоритет

## Расширение: конкурентный анализ (2026-03-19)

Парсер и анализатор применимы не только для brand voice, но и для **анализа конкурентов клиента**.

### Когда использовать

Триггеры: "проанализируй конкурентов", "что делают другие магазины/каналы", "сравни с конкурентами", запрос от клиента про чужие TG-каналы.

### Отличие от Brand Voice Clone

- Brand Voice → цель: скопировать стиль для написания
- Конкурентный анализ → цель: инсайты что работает, что не работает, незанятые ниши

### Промпт для конкурентного анализа

`tg-style-analyzer.py` генерирует brand voice prompt. Для конкурентного анализа — использовать **отдельный промпт**, переданный в Claude CLI напрямую:

```python
prompt = f"""
Ты — эксперт по маркетингу для [ниша клиента].
Проанализируй TG-каналы конкурентов и дай инсайты для [Имя клиента].

Данные канала 1: [вставить топ-посты, avg views, формат]
Данные канала 2: [...]

Формат ответа:
### 📊 Сравнительная таблица (метрики)
### 🏆 Что работает у конкурентов
### ⚠️ Слабые места — шансы для клиента
### 🚀 Топ-5 идей для канала клиента
### 📅 Контент-план на 2 недели
"""
```

### Фильтр постов в парсере

При `--limit 50` реальное кол-во постов может быть существенно меньше (фильтр `len(text) < 30` отсекает media-only и короткие посты). Нормально, если из 50 осталось 16 — это содержательные посты.

### Отправка результата клиенту

После анализа — отправить в HQ-группу клиента через бота (не через userbot). Разбить на 5-6 сообщений: шапка + сравнение + что работает + слабые места + идеи + контент-план.
