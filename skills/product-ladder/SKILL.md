# Product Ladder — Проектирование продуктовой лестницы

## Когда использовать
- Клиент продаёт только 1 продукт и теряет 97% трафика
- Нужно спроектировать тарифную сетку (3 тарифа)
- Нужна декомпозиция финансовой цели (сколько трафика нужно?)
- Запуск нового продукта — определить всю линейку
- Аудит существующей продуктовой линейки на пробелы

### HARD GATES
- **Продуктовая лестница без декомпозиции = FAIL** (лестница без цифр — фантазия)
- **Декомпозиция без основного продукта = FAIL** (нельзя считать воронку без ядра)
- **Менее 3 уровней в лестнице = FAIL** (минимум: лид-магнит + основной + максимизатор)

## Зачем

Сбалансированная продуктовая линейка — это ядро воронки. Без неё бизнес работает только с горячими клиентами (2-3% трафика), теряя 97% потенциальной выручки. Линейка позволяет:
- Конвертировать холодных → тёплых → горячих
- Зарабатывать больше на одном клиенте (LTV)
- Тратить больше на привлечение, обгоняя конкурентов

## Workflow (5 фаз)

### Phase 1: Определить основной продукт

**Вопрос:** Какой продукт связан с ключевыми компетенциями и приносит >60% выручки?

- Основной продукт — тот, ради продажи которого создавался бизнес
- Если несколько основных продуктов решают разные проблемы — для каждого строить свою воронку
- Главная задача: решить проблему/потребность, с которой клиент обращается

**Пример:** Цветочный магазин: букеты из роз (300к/мес) = 67% выручки → это основной продукт.

### Phase 2: Построить лестницу вверх и вниз

От основного продукта строим ступени в обе стороны:

```
┌─────────────────────────────────────────────────┐
│  5. УДЕРЖАНИЕ (рекурринг)                       │  ← возвращать снова и снова
│  4. МАКСИМИЗАТОРЫ (5 типов)                     │  ← заработать больше без доп. затрат
│  3. ОСНОВНОЙ ПРОДУКТ                            │  ← ядро бизнеса, >60% выручки
│  2. ТРИПВАЙР (1-10% от основного)               │  ← первая покупка, точка невозврата
│  1. ЛИД-МАГНИТ (бесплатно)                      │  ← контакты, вход в воронку
└─────────────────────────────────────────────────┘
```

#### Уровень 1: Лид-магнит
Бесплатный, быстрый в потреблении (5-15 мин) продукт с высокой ценностью → контакты.

**5 критериев лид-магнита:**
1. **Польза** — релевантен проблеме клиента, помогает сделать первый шаг
2. **Быстрое потребление** — 5-15 минут (НЕ книга, НЕ видеокурс)
3. **Специфичность** — чем уже проблема, тем выше конверсия
4. **Желание поделиться** — красиво оформлен, хочется отправить друзьям
5. **Создаёт потребность** — в основном продукте

**НЕ делать лид-магнитом:** рассылку (растянутый непродукт), купон на скидку (обязывает к покупке до доверия).

#### Уровень 2: Трипвайр
Продукт с низкой ценой и высокой воспринимаемой ценностью → первая денежная итерация.

- Цена: **1-10% от основного продукта** (ниже порога принятия решения)
- Задача: научить клиента покупать у вас, НЕ заработать прибыль
- НЕ урезанная версия основного (убьёт потребность в покупке)

**6 типов трипвайров:** ознакомительная услуга, мини-курс, вспомогательные PDF-материалы, пробная версия, пробная партия, популярный товар по спеццене.

#### Уровень 3: Основной продукт
Ядро бизнеса — то, что решает главную проблему клиента.

#### Уровень 4: Максимизаторы прибыли (5 типов)

| # | Тип | Суть | Пример |
|---|-----|------|--------|
| 1 | **Дополнительные опции** | Модификация, улучшающая функционал/внешний вид | iPhone: 64GB → 256GB → 512GB |
| 2 | **Более дорогая версия** | Усовершенствованная версия продукта | MacBook Air 119к → MacBook Pro 199к |
| 3 | **VIP-продукт** | Максимальный набор опций | Стандартный номер 6к → LUX 80к/ночь |
| 4 | **Бандл (комплект)** | Пакет товаров дешевле, чем по отдельности | Сноуборд + крепление + ботинки: 65к вместо 82к |
| 5 | **Ассоциированный продукт** | То, что обычно покупают вместе | Шлем при покупке мотоцикла |

**Направляющие вопросы:**
- Какие дополнительные опции/продукты можно предложить к основному?
- Какой альтернативный продукт с меньшей стоимостью, но высокой маржой предложить тем, кто не готов к основному?

#### Уровень 5: Продукты на удержание
Решают постоянную (хроническую) потребность клиента, которая возникает снова и снова.

**Виды:**
- Абонемент / подписка (доступ на срок)
- Сервисное обслуживание (регулярная поддержка)
- Страховка / гарантия (снижение рисков)
- Расходники / комплектующие (необходимы для использования основного)

### Phase 3: Концепт продукта (по Тимочко, 8 шагов)

Для каждого продукта в линейке (особенно основного) заполнить:

| Шаг | Вопрос | Пример |
|-----|--------|--------|
| 1 | **Для кого?** | 4 сегмента ЦА (сегмент + описание + результат) |
| 2 | **Большое обещание** | "[Результат] за [срок] без [болей]" |
| 3 | **По итогу** | Конкретные артефакты/результаты |
| 4 | **Программа** | По модулям с результатом после каждого |
| 5 | **Формат** | Сколько длится? Какие форматы поддержки? Личное вовлечение? |
| 6 | **Тарифы** | 3 тарифа (см. Phase 5: Иллюзия выбора) |
| 7 | **Wow-фишки** | 5-10 фишек (челленджи, живые встречи, чат, эфиры с приглашёнными) |
| 8 | **Болеутолители / Гарантии** | "Не придётся...", "Займёт всего...", "Вернём деньги если..." |

### Phase 4: Декомпозиция цели (Revenue = P x Q)

Формула: **Выручка = Средний чек x Количество продаж**

Алгоритм "шаг назад" от желаемой выручки:

```
Желаемая чистая прибыль
  ÷ % прибыли от выручки
= Необходимая выручка
  ÷ средний чек
= Необходимое количество продаж
  ÷ конверсия отдела продаж (20-50%)
= Необходимое количество заявок
  ÷ конверсия сайта/воронки (2-7%)
= Необходимый трафик (посетители)
  ÷ 30 дней
= Дневная активность (посетители/день, заявки/день, продажи/день)
```

**Пример расчёта:**
- Цель: 1 000 000 руб. чистыми
- Рентабельность: 11% → нужно 9 000 000 руб. выручки
- Средний чек: 30 000 руб. → нужно 300 продаж
- Конверсия ОП: 25% → нужно 1 200 заявок
- Конверсия сайта: 2% → нужно 60 000 посетителей
- **В день:** 2 000 посетителей → 40 заявок → 10 продаж → 300 000 руб./день
- Рекламный бюджет: ~750 000 руб./мес (12,5 руб./посетитель)

**Принципы:**
- "Похоже на правду" — примерные цифры лучше, чем их отсутствие
- "Принцип существенности" — берите 1-2 направления, приносящие основную прибыль
- "Принцип разрезания" — нарежьте выручку по категориям товаров/услуг
- Считать средним по году (без сезонных пиков)

---

### Phase 5: Иллюзия 3 тарифов

**Правило:** это НЕ 3 разных продукта. Это 1 продукт в 3 комплектациях. Цель — 80% покупок на центральный тариф.

| Тариф | Стратегия | Психология |
|-------|-----------|------------|
| **Минус** | Убрать критичную фичу за незначительную скидку | "Ну что мне жалко было доплатить..." → сожаление, апгрейд |
| **Центр** | Оптимальный набор — всё что нужно | Целевой: 80% покупок должны быть здесь |
| **Плюс** | Добавить VIP/индивидуальное за значительно большую цену | Якорь цены: делает центральный "разумным" |

**Порядок проектирования:**
1. Сначала проектируйте ЦЕНТРАЛЬНЫЙ тариф (всё, что хотите продать)
2. Затем создайте МИНУС (уберите важное, снизьте цену немного)
3. Затем создайте ПЛЮС (добавьте VIP-бонусы, поднимите цену значительно)

**Пример (онлайн-курс):**
- База ("Подглядеть"): 9 900 руб. — доступ к 3 неделям, чат, разборы как зритель
- Оптимум: 19 900 руб. — + бонусная неделя, проверка ДЗ, эфиры с куратором
- Премиум: 169 000 руб. — + живые занятия, личный куратор, диплом, 6 мес. доступ

## Anti-patterns

| Ошибка | Почему плохо | Решение |
|--------|-------------|---------|
| Только 1 продукт | Теряете 60-80% потенциальной выручки | Построить минимум 3 уровня (лид-магнит + основной + максимизатор) |
| Трипвайр дороже 5-10% от основного | Не импульсная покупка, барьер не снижается | Держать 1-10% от цены основного |
| Лид-магнит без ценности | Скачают и забудут, не создаёт доверия | 5 критериев: польза + быстро + специфично + поделиться + потребность |
| Нет рекурринга | Каждый месяц ищете новых клиентов | Добавить подписку/абонемент/расходники |
| 3 тарифа = 3 разных продукта | Размывается фокус, нет иллюзии выбора | 1 продукт в 3 комплектациях |
| Тариф-минус с большой скидкой | Нет сожаления, люди покупают дешёвый и довольны | Скидка незначительная, убрана критичная фича |
| Декомпозиция без цифр | "Хочу много зарабатывать" — не план | Конкретно: выручка → продажи → заявки → трафик → действия/день |
| Лид-магнит = видеокурс | Слишком долго потреблять (нужно 5-15 мин) | PDF, чеклист, калькулятор |
