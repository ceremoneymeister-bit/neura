# Яндекс Дзен — полный скилл для работы с платформой

## Назначение

Скилл для создания, публикации и монетизации контента на Яндекс Дзен. Включает: написание статей, упаковку каналов, стратегию контента, аналитику, генерацию обложек и автопостинг. Основан на методологии Игоря Градова (сеть 115+ каналов, оборот 6.8 млн ₽/мес).

## Когда использовать

**Триггеры:** Дзен, Яндекс Дзен, Zen, dzen, статья для Дзена, канал на Дзене, монетизация Дзена, обложка для Дзена, контент-план Дзен, алгоритм Дзена, RPM, дочитываемость.

**НЕ использовать:** для Telegram-постов (→ channel-autopilot), для VK (→ vk-community), для Instagram (→ social-content).

## Режимы работы

### Режим A: Написание статьи
**Триггер:** «напиши статью для Дзена», «статья на Дзен», «контент для Дзена»

1. **Контекст** — уточни: тема, ниша канала, целевая аудитория
2. **Прочитай** `references/article-writing.md` + подходящий шаблон из `templates/`
3. **Заголовки** — сгенерируй 3-5 вариантов (50-60 символов, без кликбейта, с интригой)
   - Интегрируй `headline-lab` если нужно больше вариантов
4. **Напиши статью** по правилам:
   - 3000-5000 символов
   - Хук в первом абзаце (вопрос / факт / история)
   - Подзаголовки каждые 300-500 символов
   - Минимум 3-4 изображения
   - Нет внешних ссылок в первых 1000 символах
   - Уникальность текста ≥ 85%
5. **Проверка на AI-маркеры** — убери фразы из чёрного списка (см. `references/article-writing.md`)
6. **Обложка** — через `scripts/zen-cover-generator.py` или `nano-banana-pro`
   - Размер: 1920x1080 (16:9), safe zone 1440x1080
7. **Скоринг** — `scripts/zen-article-scorer.py` (целевой балл ≥ 7/10)
8. **Гейт:** если скор < 7 → доработай и пересчитай

### Режим B: Упаковка канала
**Триггер:** «упакуй канал на Дзене», «создай канал на Дзен», «настрой канал»

1. **Прочитай** `references/channel-packaging.md`
2. **Выбор ниши** — помоги выбрать по критериям:
   - Наличие внешних рекламодателей
   - Конкуренция в нише
   - Потенциал масштабирования
   - Сезонность
3. **Название канала** — 3-5 вариантов (SEO-ключевики + запоминаемость)
4. **Описание** — 2-3 предложения с ключевыми словами
5. **Аватар** — через `nano-banana-pro` (128x128+, яркий, читаемый)
6. **Контент-план старта** — первые 10-15 статей по шаблонам из `templates/`
7. **Чеклист** — пройди `templates/channel-setup-checklist.md`

### Режим C: Стратегия контента
**Триггер:** «стратегия для Дзена», «контент-план Дзен», «как раскрутить канал»

1. **Прочитай** `references/content-strategy.md` + `references/algorithm-mechanics.md`
2. **Анализ ниши** — оцени конкуренцию, сезонность, RPM в нише
3. **Контент-календарь** — по шаблону `templates/content-calendar-template.md`
   - Частота: 1-3 статьи/день (оптимум для роста)
   - Микс форматов: статьи + посты + нарративы
4. **Cross-platform** — Дзен → Telegram воронка (статья → лид-магнит → подписка)
5. **Прочитай** `references/monetization.md` для целей по доходу

### Режим D: Публикация
**Триггер:** «опубликуй на Дзен», «выложи статью», «запостить на Дзен»

1. **Проверь cookies** — `.secrets/yandex_cookies.json` (Playwright авторизация)
2. **По умолчанию → черновик** (`--draft`). Живая публикация только по явной команде
3. Запусти `scripts/zen-publish.py`:
   ```bash
   python3 .agent/skills/yandex-zen/scripts/zen-publish.py \
     --title "Заголовок" --body article.md --cover cover.jpg --draft
   ```
4. **Скриншот превью** перед публикацией
5. **Вернуть URL** опубликованной статьи

### Режим E: Аналитика
**Триггер:** «аналитика Дзена», «статистика канала», «как идут дела на Дзене»

1. **Прочитай** `references/analytics-guide.md`
2. Запусти `scripts/zen-analytics.py` (Playwright, скрейпинг)
3. **Интерпретация метрик:**
   - CTR: 10-15% норм, 15-30% отлично, <5% — проблема с заголовком/обложкой
   - Дочитываемость: ≥80% цель, <40% критично
   - RPM: зависит от ниши, ≥100₽ хорошо
4. **Рекомендации** — что улучшить, какие статьи масштабировать

### Режим F: Справка / База знаний
**Триггер:** «как работает алгоритм Дзена», «что такое RPM», вопросы про Дзен

1. Определи, какой reference-файл отвечает на вопрос
2. Прочитай его и дай структурированный ответ
3. Если вопрос не покрыт базой → дополни через WebSearch

## Ключевые правила 2026 года

> Из `references/algorithm-mechanics.md`:
- Рекомендации сократили в **6-15 раз** — теперь не главный источник трафика
- Основной трафик = **лента + поиск Яндекса**
- Формула роста: подписчики × частота публикаций × качество
- AI-тексты без редактуры = бан в рекомендациях
- Заголовок ≤ 60 символов, текст ≥ 85% уникальности
- Обложка: яркая, контрастная, без мелких деталей, 16:9

## Антипаттерны

1. **AI без редактуры** — Дзен детектит фразы «в мире где», «в заключение», «давайте разберёмся». Список в `references/anti-patterns.md`
2. **Кликбейт** — в 2026 = показ только подписчикам, смерть охватов
3. **Внешние ссылки в начале** — алгоритм пессимизирует
4. **Копипаст** — уникальность < 85% = бан в рекомендациях
5. **Нерегулярность** — < 3 статей/неделю = потеря позиций в ленте
6. **Игнор аналитики** — публиковать без анализа CTR/дочитываемости = работать вслепую
7. **Стоковые фото** — Дзен любит уникальные изображения
8. **Статья < 2000 символов** — не хватает глубины для рекомендаций

## Стандарты качества

Перед публикацией КАЖДАЯ статья проходит чеклист:
- [ ] Заголовок: 50-60 символов, ключевое слово в начале, без кликбейта
- [ ] Лид: первые 2 предложения отвечают на «зачем читать»
- [ ] Объём: 3000-5000 символов
- [ ] Подзаголовки: каждые 300-500 символов
- [ ] Изображения: обложка + 3-4 в тексте, все уникальные
- [ ] Уникальность: ≥ 85% (text.ru или аналог)
- [ ] AI-маркеры: убраны все фразы из чёрного списка
- [ ] Внешние ссылки: нет в первых 1000 символах
- [ ] SEO: ключевое слово в заголовке + первом абзаце
- [ ] Скор: ≥ 7/10 по zen-article-scorer
