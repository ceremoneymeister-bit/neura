# quiz-funnel

Проектирование квиз-воронок для квалификации лидов и увеличения конверсии. Квиз = интерактивный пошаговый опросник, где человек отвечает на простые вопросы и в конце оставляет контакты в обмен на бонус. Конверсия квиза: 15-40% vs 2-5% у обычных форм.

Источник: НЕЧ20 / ConversionArt (Иван Дербилов) — kvizy-2.0.

---

## Когда использовать

- Квиз на лендинге (как CTA первого экрана или отдельный блок)
- Квиз-бот в Telegram (inline keyboards для ответов)
- Квиз для сегментации аудитории (распределение заявок по менеджерам/группам)
- Квиз для сбора контактов (подписная база + квалификация)
- Квиз как отдельная рекламная страница (для A/B тестов с холодным трафиком)
- Квиз-калькулятор (расчёт стоимости + лидогенерация)

**CTA Stress:** Квиз по своей природе low-stress CTA. Отвечать на вопросы психологически легче, чем заполнять форму заявки или звонить. Поэтому квиз идеален для холодного трафика. Два механизма работают в его пользу:
1. **Обмен информации** — человек понимает, что чем больше расскажет, тем точнее получит ответ (в отличие от формы "имя + телефон", где его просто будут допрашивать)
2. **Незакрытый гештальт** — начав квиз (особенно если Q1 простой), человек стремится дойти до конца

## 5 типов квизов

### 1. Расчётный (калькулятор)
**Оффер:** "Рассчитайте стоимость..."
**Когда:** Продукт с вариативной ценой (ремонт, строительство, услуги, производство).
**Пример:** "Рассчитайте стоимость чип-тюнинга и прирост мощности вашего автомобиля"

### 2. Подбор (рекомендация)
**Оффер:** "Подберём идеальный вариант..."
**Когда:** Широкий ассортимент, клиент не знает что выбрать.
**Пример:** "Бесплатно подберите идеальный проект вашего будущего дома всего за 3 минуты"

### 3. Комбинированный (подбор + расчёт)
**Оффер:** "Подберём + рассчитаем стоимость..."
**Когда:** Нужно определить потребность И сориентировать по цене.
**Пример:** "Подберите модель поломочной машины и узнайте её стоимость"

### 4. Тестовый (викторина)
**Оффер:** "Проверьте свои знания..." / "Узнайте свой уровень..."
**Когда:** Образовательные продукты, экспертные ниши, геймификация.
**Пример:** "Викторина на знание сортов вин — получите скидку по результату"
**Фишка:** Каждому ответу можно присвоить баллы, по итогу — персонализированный результат + скидка.

### 5. Лидмагнитный (короткий)
**Оффер:** "Получите бесплатный [материал]..."
**Когда:** Сбор подписной базы, нужна минимальная квалификация (1-2 вопроса) перед выдачей файла.
**Пример:** "Ответьте на 2 вопроса и получите подходящий пакет бонусов"

### Как выбрать тип
```
Клиент знает, чего хочет? → Расчётный
Клиент НЕ знает, чего хочет? → Подбор
Нужно и то, и другое? → Комбинированный
Образование / экспертиза? → Тестовый
Нужна только база? → Лидмагнитный
```

## Архитектура вопросов — Workflow

### Phase 1: Определить цель квиза

```
ЦЕЛЬ = одна из:
  - Сбор контактов (подписная база)
  - Квалификация лидов (приоритизация заявок по менеджерам)
  - Сегментация (разные офферы разным группам)
  - Расчёт стоимости (автоматическое КП)
  - Комбинация

ТИП = выбрать из 5 типов выше
БОНУС = что получит человек за прохождение (файл, расчёт, скидка, каталог)
```

### Phase 2: Спроектировать 5-7 вопросов

**Золотое правило:** Задача квиза — получить заявки, а НЕ анкетировать клиента. Каждый вопрос должен быть оправдан. Если вопрос ради вопроса — безжалостно удалять. Лучше 2 вопроса с конверсией 10%, чем 10 вопросов с конверсией 1%.

#### Структура вопросов:

| # | Тип вопроса | Цель | Пример |
|---|-------------|------|--------|
| Q1 | Потребность (широкий, лёгкий) | **Вовлечение.** На такой вопрос легко ответить, он определяет, что клиент хочет. | "Какой стиль кухни вы предпочитаете? Современный / Неоклассика / Лофт" |
| Q2 | Уточнение потребности | Детализация. Параметры продукта или необходимый функционал. | "Какой материал фасада вам нравится? ДСП / МДФ / Рамочный МДФ" |
| Q3 | Объём / комплектация | **Квалификация: размер сделки.** Выявляет крупных заказчиков. | "Укажите площадь кровли: до 50 м2 / 50-100 м2 / более 100 м2" |
| Q4 | Бюджет | **Квалификация: платёжеспособность.** Указывать даже диапазон, с которым не работаете — для фильтрации нецелевых. | "Какой бюджет на свадьбу? До 300к / 300-500к / 500к-1М / Более 1М" |
| Q5 | Сроки | **Квалификация: срочность.** Определяет приоритетность обработки. | "Когда должна быть установлена кухня? В течение месяца / 3 месяцев / полугода" |
| Q6 | Доп. потребности / подарок | Повышение среднего чека или лояльности. | "Какой подарок к заказу? Скидка 50% на чистку / Аксессуары / Набор по уходу" |
| Q7 | Контактная форма | **Захват лида.** | Имя + телефон (минимум). Email — только если нужна подписная база. |

#### Типы вариантов ответа (от лучшего к худшему по конверсии):
1. Картинки + текст (максимальная вовлечённость)
2. Текстовые варианты (стандарт)
3. Диапазон / ползунок (для числовых значений)
4. Выпадающий список (для длинных списков)
5. Свободное поле (ОПАСНО — требует усилий, снижает конверсию)
6. Календарь/дата (ОПАСНО — только если клиент точно знает дату)

#### Правила составления вопросов:
- Всегда добавлять вариант "Другое" / "Не знаю" / "Ещё не определился" — чтобы человек мог продолжить, даже если не знает точный ответ
- Не использовать профессиональные термины — писать словами клиента
- Вариативность (branching): ответ на Q1 определяет следующие вопросы (мужская одежда → ветка про костюмы, женская → ветка про платья)
- Каждый вопрос может сопровождаться "Консультантом" — фото + имя + подсказка рядом с вопросом (повышает доверие)

### Phase 3: Результирующая страница

**Hard gate: Квиз без результирующей страницы = FAIL.**

Результат должен быть **персонализированным** на основе ответов:
- Расчётный квиз → предварительная стоимость
- Подбор → конкретная рекомендация (модель, вариант, пакет)
- Тестовый → балл + интерпретация
- Комбинированный → рекомендация + цена

Показывать результат рекомендуется **после** заполнения формы контактов (повышает конверсию в заявку).

### Phase 4: Страница "Спасибо" (Thank You Page)

**Hard gate: Страница "Спасибо" = тупик = FAIL.** Это НЕ мёртвая заглушка "Спасибо, ожидайте звонка".

Обязательные элементы:
1. **Благодарность** — "Спасибо за ответы!"
2. **Ожидание** — что будет дальше ("Менеджер свяжется в течение 15 минут")
3. **Прогрев** — видео о компании / кейсы / портфолио (человек ждёт звонка и знакомится с вами)

Продвинутые элементы (выбрать 1-2):
- **OTO (One Time Offer)** — специальное предложение, доступное только сейчас
- **Бонус** — ссылка на скачивание обещанного материала
- **Соцсети** — подписка на Instagram / Telegram / YouTube
- **Следующий шаг** — запись на консультацию / демо / визит

## Посадочная страница квиза

### Элементы первого экрана
1. **Заголовок (оффер)** — формула: `[Глагол в повелительном наклонении] + [что получит клиент] + [ресурсы]`
   - "Рассчитайте стоимость ремонта, ответив на 5 вопросов"
   - "Подберите идеальный проект дома за 3 минуты"
2. **Подзаголовок** — что нужно сделать + что получит в конце + бонус
   - "Ответьте на 6 вопросов и получите расчёт стоимости + PDF с плюсами и минусами + подарок"
3. **CTA-кнопка** — текст = начало заголовка (НЕ синоним! "Рассчитайте стоимость" → кнопка "Рассчитать стоимость")
4. **Изображение продукта** — реальное фото > стоковое (стоковые люди = "мы ненастоящие")
5. **Макап бонуса** — визуализация того, что получит клиент (разворот книги, экран смартфона)
6. **Логотип + контакты** — кликабельные
7. **Прогресс-бар** — показывает, сколько шагов осталось

### Форма захвата контактов
- **Благодарность** за ответы
- **Призыв к действию** — "Оставьте контакты и получите расчёт"
- **Минимум полей:** Имя + Телефон. Email — только если нужна подписная база
- **Фишка полей:** Не "Укажите телефон", а "Напишите телефон — мы закрепим за ним скидку 20%". Не "Email", а "Куда отправить каталог и прайс?"
- **Мессенджеры** — кнопки перехода в бота (альтернатива форме, повышает конверсию)
- **Скидка за прохождение:** фиксированная (5%) / тающая (уменьшается с каждым вопросом) / растущая (увеличивается с каждым ответом)

## Размещение квиза

| Вариант | Когда использовать |
|---------|-------------------|
| Отдельная страница | Быстрый запуск, нет сайта, A/B тест с рекламы |
| Блок на лендинге (2-й экран) | CTA первого экрана скроллит к квизу |
| Дубль перед футером | Повторный CTA для тех, кто долистал |
| Виджет на главном экране | Привлечь внимание без перехода |
| Popup (exit-intent / по таймеру) | Спасение уходящего трафика |
| Внутри VK-сообщества | Клиент проходит квиз не выходя из соцсети |
| Telegram-бот | Inline keyboards для ответов, бот = CRM |

## Квиз в Telegram

Формат реализации через бота с inline keyboards:
1. Каждый вопрос = сообщение + inline-кнопки с вариантами
2. Ответ пользователя сохраняется в БД
3. Прогресс показывается в тексте сообщения ("Вопрос 3 из 6")
4. После последнего вопроса — запрос контактов (кнопка "Поделиться номером")
5. Результат — персонализированное сообщение на основе ответов
6. Follow-up — автоматическое сообщение через N часов если не было конверсии

**Преимущество TG-квиза:** клиент сразу в боте, не нужна отдельная форма захвата — контакт уже есть.

## Метрики и кейсы

### Бенчмарки
| Метрика | Обычная форма | Квиз |
|---------|---------------|------|
| Конверсия в заявку | 2-5% | 15-40% |
| Стоимость заявки | Высокая | x2-5 ниже |
| Качество лида | Низкое (только имя+телефон) | Высокое (квалифицированный) |

### Кейсы (из НЕЧ20 / ConversionArt)
- **Standard Prodmage** (пищевые производства): 19k руб. бюджет → 74 квалифицированных заявки на 118.4 млн руб. за 30 дней
- **Invite** (организация свадеб): 10k/мес → 100-200 заявок, 50-70 руб/заявка. 60 свадеб за лето, средний чек 500к руб.

## Anti-patterns

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Слишком много вопросов (>10) | Drop-off растёт экспоненциально | 5-7 вопросов оптимально |
| Нет прогресс-бара | Человек не знает, сколько ещё | Всегда показывать "Шаг X из Y" |
| Контактная форма без объяснения зачем | "Зачем им мой телефон?" → уход | "Оставьте контакты — мы отправим расчёт" |
| Результат без персонализации | Одинаковый ответ всем = обман | Разные результаты на основе ответов |
| Страница "Спасибо" = тупик | Потерянная возможность прогреть/продать | OTO, видео, соцсети, следующий шаг |
| Синоним в кнопке | "Рассчитайте стоимость" → кнопка "Узнать цену" — путает | Текст кнопки = начало заголовка |
| Профессиональные термины в вопросах | Клиент не понимает → уходит | Писать словами клиента |
| Свободные поля вместо вариантов | Высокое трение → низкая конверсия | Варианты + "Другое" |
| Стоковые фото с иностранцами | "Мы ненастоящие" | Реальные фото продукта/команды |
| Нет варианта "Не знаю" | Человек застревает → уходит | Всегда добавлять escape-вариант |
