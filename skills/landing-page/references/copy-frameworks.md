# Copy Frameworks — Формулы и паттерны копирайтинга

## 10 формул заголовков

### 1. Результат + Срок
**Формула:** "[Результат] за [срок]"
**Пример:** "AI-отдел из 6 агентов за 24 часа"

### 2. Вопрос-боль
**Формула:** "[Вопрос, который задаёт ЦА]?"
**Пример:** "Устали тратить 40 часов в месяц на рутину?"

### 3. Числовой якорь
**Формула:** "[Число] [сущность] для [результат]"
**Пример:** "113 навыков для автоматизации вашего бизнеса"

### 4. Сравнение-контраст
**Формула:** "Не [старый способ]. [Новый способ]."
**Пример:** "Не чат-бот. Полноценный AI-сотрудник."

### 5. Social Proof заголовок
**Формула:** "[N клиентов] уже [результат]"
**Пример:** "325 предпринимателей уже автоматизировали рутину"

### 6. Провокация
**Формула:** "[Шокирующий факт / контринтуитивное утверждение]"
**Пример:** "Ваш лучший сотрудник не спит, не болеет и стоит дешевле стажёра"

### 7. Трансформация
**Формула:** "Из [состояние А] в [состояние Б]"
**Пример:** "Из хаоса задач — в систему, которая работает сама"

### 8. Минимизация усилий
**Формула:** "[Результат] без [неприятность]"
**Пример:** "Контент для соцсетей без часов на копирайтинг"

### 9. Как это работает
**Формула:** "Как [ЦА] [достигает результата] с помощью [продукт]"
**Пример:** "Как предприниматели экономят 40 часов в месяц с AI-агентом"

### 10. Уникальный механизм
**Формула:** "[Продукт]: [уникальный механизм / метафора]"
**Пример:** "Нейра: персональный AI-отдел, который работает 24/7"

## 8 паттернов CTA-копирайтинга

### 1. Action + Benefit
**Паттерн:** "[Глагол] + [что получит]"
**Примеры:** "Начать экономить время", "Получить AI-агента"

### 2. First Person
**Паттерн:** "Я хочу [результат]" / "Да, хочу [результат]"
**Примеры:** "Да, хочу попробовать", "Мне нужен AI-агент"

### 3. Low Friction
**Паттерн:** "[Действие] — бесплатно / без [барьер]"
**Примеры:** "Попробовать бесплатно", "Без привязки карты"

### 4. Urgency
**Паттерн:** "[Действие] + [ограничение]"
**Примеры:** "Забрать место — осталось 5", "Успеть до повышения цены"

### 5. Specific Value
**Паттерн:** "[Действие] за [конкретная цена / условие]"
**Примеры:** "Начать за 7 000 ₽/нед", "30 минут — бесплатно"

### 6. Risk Reversal
**Паттерн:** "[Действие] + [гарантия]"
**Примеры:** "Попробовать — не подойдёт, вернём деньги", "7 дней без риска"

### 7. Micro-commitment
**Паттерн:** "[Маленький первый шаг]"
**Примеры:** "Просто посмотреть демо", "Задать вопрос в Telegram"

### 8. Social CTA
**Паттерн:** "[Действие] + [присоединиться к группе]"
**Примеры:** "Присоединиться к 325 предпринимателям", "Стать частью клуба"

## Типы доказательств

### Quantitative (самые сильные)
**Что:** Конкретные числа и метрики результатов.
**Формат:** "X% рост / Y часов экономии / Z₽ прибыли"
**Пример:** "ROI 340% за первый месяц использования"
**Когда:** Если есть реальные данные — всегда ставить первым.

### Testimonials
**Что:** Цитаты реальных клиентов с именем, фото, должностью.
**Формат:**
```
"[Конкретный результат в 1-2 предложениях]"
— Имя Фамилия, должность, компания
```
**Правила:**
- Фото обязательно (повышает доверие на 35%)
- Конкретика > абстракция ("экономлю 10 часов" > "очень удобно")
- 3-5 отзывов оптимально (> 7 = снижение внимания)

### Case Studies
**Что:** Развёрнутая история: ситуация → решение → результат.
**Формат:**
```
Проблема: [в чём была боль]
Решение: [что конкретно сделали]
Результат: [измеримый outcome]
```
**Когда:** B2B, высокий чек, сложный продукт.

### Authority
**Что:** Внешнее подтверждение экспертности.
**Форматы:**
- Сертификаты и награды
- Публикации в медиа
- Партнёрства с известными брендами
- Количественные показатели ("На рынке 13 лет")

## Фреймворк обработки возражений (A-R-E-C)

Для FAQ-секции и любых точек сомнения:

### 1. Acknowledge — Признать
Не спорить с возражением. Показать, что понимаем.
> "Да, это логичный вопрос. Многие спрашивают о стоимости."

### 2. Reframe — Переформулировать
Изменить рамку восприятия. Показать с другой стороны.
> "Если посчитать стоимость в пересчёте на час — это дешевле любого сотрудника."

### 3. Evidence — Доказать
Факт, кейс, цитата клиента, гарантия.
> "Алексей из [компания] окупил подписку за 3 дня: 'Агент за ночь написал 12 постов, которые раньше занимали неделю.'"

### 4. CTA — Вернуть к действию
Мягкий переход обратно к целевому действию.
> "Попробуйте бесплатно 7 дней и убедитесь сами."

## Feature → Benefit → Outcome цепочки

**Принцип:** Пользователь покупает не Feature, а Outcome. Feature — это средство, Outcome — трансформация.

### Структура

```
Feature:  [Что есть в продукте — объективный факт]
     ↓
Benefit:  [Что это даёт пользователю — практическая выгода]
     ↓
Outcome:  [Как меняется жизнь/бизнес — эмоциональный результат]
```

### Примеры

```
Feature:  AI-агент работает 24/7
Benefit:  Задачи выполняются даже когда вы спите
Outcome:  Вы просыпаетесь с готовым контент-планом на неделю

Feature:  113 встроенных навыков
Benefit:  Не нужно нанимать специалистов на каждую задачу
Outcome:  Один инструмент заменяет отдел из 5 человек

Feature:  Telegram-интеграция
Benefit:  Управление прямо из мессенджера, без новых приложений
Outcome:  Работа с AI так же естественна, как переписка с коллегой
```

### Правила
1. Каждая фича на лендинге ДОЛЖНА иметь полную цепочку F→B→O
2. В заголовке карточки — Benefit или Outcome (НЕ Feature)
3. Feature — мелким текстом или тегом
4. На одну фичу — одна цепочка (не смешивать)
5. Outcome должен быть эмоционально резонирующим для ЦА
