# Conversion Architecture — Анатомия лендинга

## 9 секций лендинга

Стандартная последовательность. Не все секции обязательны — выбор зависит от awareness-уровня аудитории и типа оффера.

### 1. Hero

**Задача:** За 3 секунды ответить на "Что это?", "Для кого?", "Почему мне это нужно?"

**Компоненты:**
- Заголовок (H1) — главное обещание / провокация / трансформация
- Подзаголовок — расшифровка: что именно получит пользователь
- CTA-кнопка — одно целевое действие
- Визуал — продукт / результат / социальное доказательство
- Social proof snippet — "500+ клиентов" / логотипы / рейтинг

**Варианты Hero по awareness:**

| Awareness | Hero-стратегия | Пример заголовка |
|-----------|---------------|-----------------|
| Unaware | Провокация / история / шок-факт | "87% предпринимателей теряют 40 часов в месяц на рутину" |
| Problem-aware | Эмпатия + намёк на решение | "Устали тратить вечера на задачи, которые можно автоматизировать?" |
| Solution-aware | Дифференциация | "AI-агент, который работает как сотрудник, а не как чат-бот" |
| Product-aware | Оффер + доказательство | "Нейра — AI-отдел из 6 агентов. 113 навыков. 24/7." |
| Most-aware | CTA + цена + срочность | "Тест-драйв 7 дней — 7 000 ₽. Осталось 3 места." |

### 2. Social Proof Bar

**Задача:** Снять первичное недоверие сразу после Hero.

**Форматы:**
- Логотипы клиентов (5-7 штук)
- Цифры: "500+ клиентов", "3 года на рынке", "98% довольны"
- Мини-цитата от известного клиента
- Рейтинг со звёздами

**Когда пропустить:** Стартап без клиентов → заменить на "Как это работает" (видео/анимация)

### 3. Pain / Problem

**Задача:** Обострить проблему, которую решает продукт. Пользователь должен узнать себя.

**Паттерны:**
- "Знакомо?" + список болей (3-5 пунктов)
- "До / После" — контраст текущей ситуации и желаемой
- История клиента с аналогичной болью
- Визуальная метафора (хаос → порядок)

**Психология:** Loss Aversion (потери от бездействия > выгоды от действия)

### 4. Solution

**Задача:** Представить продукт как решение описанной проблемы.

**Структура:**
- Мост от боли к решению: "Именно поэтому мы создали X"
- 1-2 предложения: что это и как работает
- Ключевая дифференциация: почему X, а не Y
- Визуал продукта / интерфейс / схема

### 5. Benefits (не Features!)

**Задача:** Показать выгоды через цепочку Feature → Benefit → Outcome.

**Формат F→B→O:**
```
Feature: AI-агент работает 24/7
Benefit: Вы не тратите время на рутинные задачи
Outcome: Освобождается 40+ часов в месяц на стратегию и рост
```

**Количество:** 3-6 ключевых выгод. Каждая — карточка или блок.

**Визуал:** Иконки, bento grid, timeline — см. design-patterns.md.

### 6. How It Works

**Задача:** Снять страх сложности. Показать процесс в 3-5 шагах.

**Формат:**
1. Шаг 1: [Действие пользователя]
2. Шаг 2: [Что происходит]
3. Шаг 3: [Результат]

**Визуал:** Numbered steps, timeline, process diagram.

**Когда подробнее:** Сложный продукт, B2B, высокий чек — больше деталей на каждом шаге.

### 7. Proof / Testimonials

**Задача:** Доказать через third-party evidence.

**Типы доказательств (по силе):**
1. **Количественные** — "ROI 340%", "Сэкономили 120 часов", "Выручка ×3"
2. **Кейсы** — конкретная история: проблема → решение → результат
3. **Отзывы** — цитаты с фото и именем (не анонимные!)
4. **Авторитет** — сертификаты, публикации, партнёрства, логотипы

**Формат отзыва:**
```
"[Цитата с конкретным результатом]"
— Имя Фамилия, должность, компания
```

### 8. FAQ

**Задача:** Обработать возражения, которые мешают конверсии.

**Фреймворк обработки возражений:**
1. **Acknowledge** — "Понимаем ваше беспокойство о..."
2. **Reframe** — переформулировать как преимущество
3. **Evidence** — конкретный факт / кейс / гарантия
4. **CTA** — мягкий возврат к действию

**Стандартные возражения:**
- Цена слишком высока → Сравнение с альтернативами, ROI
- Не уверен что подойдёт → Тест-драйв / гарантия возврата
- Сложно разобраться → "How It Works" в 3 шага
- Нет времени сейчас → Срочность / ограниченное предложение
- Уже пробовал подобное → Дифференциация

### 9. Final CTA

**Задача:** Финальный призыв к действию. Самый важный CTA на странице.

**Структура:**
- Краткое резюме ценности (1-2 предложения)
- Основной CTA (кнопка) — повторяет или усиливает Hero CTA
- Снятие последнего возражения: гарантия / бесплатный тест / без привязки карты
- Контакт для вопросов (дополнительно)

Какую ментальную модель применять в какой секции:

| Ментальная модель | Лучшие секции | Как применять |
|-------------------|--------------|---------------|
| Loss Aversion | Pain, Final CTA | "Каждый день без X вы теряете Y" |
| Social Proof | Social Bar, Proof | Цифры, логотипы, отзывы |
| Anchoring | Benefits, Pricing | Сначала высокая цена → потом ваша |
| Scarcity | Hero, Final CTA | "Осталось N мест", таймер |
| Authority | Proof, FAQ | Сертификаты, партнёрства, эксперты |
| Reciprocity | Solution, How It Works | Бесплатная ценность до продажи |
| Commitment & Consistency | How It Works, CTA | Микро-шаги: сначала бесплатное, потом платное |
| Framing | Benefits, Pricing | "Не 40 000/мес, а 1 333/день — дешевле стажёра" |
| Paradox of Choice | CTA, Pricing | Один CTA, максимум 3 тарифа |
| Peak-End Rule | Hero, Final CTA | Сильное начало + сильный финал |
| Endowment Effect | How It Works | "Ваш персональный агент уже создан" |
| Mere Exposure | Social Bar, Proof | Повторение бренда, логотипа, цвета |
| Zeigarnik Effect | Solution | Тизер → "Узнайте больше" |
| IKEA Effect | How It Works | Вовлечение: "настройте под себя" |
| Bandwagon | Social Bar | "325 предпринимателей уже используют" |

## CTA-стратегии по типам

### Purchase (прямая продажа)
- Primary: "Купить за X ₽" / "Оформить подписку"
- Urgency: "Цена растёт через N дней"
- Guarantee: "Не подойдёт — вернём деньги за 14 дней"

### Lead Magnet (сбор контактов)
- Primary: "Скачать бесплатно" / "Получить чек-лист"
- Value: "37 страниц практических советов"
- Low friction: Только email, без телефона

### Free Trial (тест-драйв)
- Primary: "Попробовать 7 дней бесплатно"
- No risk: "Без привязки карты"
- Conversion: После триала → email-серия с кейсами

### Consultation (запись на звонок)
- Primary: "Записаться на бесплатную консультацию"
- Time: "30 минут, без обязательств"
- Qualifier: "Для компаний с оборотом от X"

### Waitlist (лист ожидания)
- Primary: "Занять место" / "Быть первым"
- Exclusivity: "Первые 100 получат X бонус"
- Scarcity: Прогресс-бар заполнения

## Адаптация по Awareness Level

### Unaware → Problem-aware
**Длина:** Длинная (7-9 секций). Нужно сначала обучить.
**Hero:** Провокация / шокирующий факт / история.
**Секции:** Hero → Education → Pain → Solution → Benefits → How It Works → Proof → FAQ → CTA.
**Тон:** Образовательный, без продажи в первой половине.

### Problem-aware → Solution-aware
**Длина:** Средняя (6-8 секций). Знают о проблеме, не знают о решении.
**Hero:** Эмпатия: "Знакомо? Есть выход."
**Секции:** Hero → Pain → Solution → Benefits → How It Works → Proof → FAQ → CTA.
**Тон:** Эмпатичный, переход к конкретике.

### Solution-aware → Product-aware
**Длина:** Средняя (5-7 секций). Сравнивают варианты.
**Hero:** Дифференциация: "Почему X, а не Y."
**Секции:** Hero → Social Proof → Benefits → Comparison → Proof → FAQ → CTA.
**Тон:** Конкретный, сравнительный, с доказательствами.

### Product-aware → Most-aware
**Длина:** Короткая (3-5 секций). Уже знают продукт.
**Hero:** Оффер + срочность.
**Секции:** Hero → Social Proof → Pricing → FAQ → CTA.
**Тон:** Прямой, оффер-ориентированный.
