# cross-poster v2

## 1. Purpose

Единый движок кросс-постинга: один пост (текст + медиа) адаптируется под каждую площадку и публикуется или подготавливается для ручной публикации. Скилл честно разделяет платформы на два уровня: полностью автоматизированные через API и те, для которых контент только подготавливается. Включает проактивные предложения по постингу и самообучение на основе аналитики.

## 2. When to Use

- Пользователь говорит: «опубликуй везде», «пост на все площадки», «разошли по соцсетям»
- Запрос на публикацию одного контента на несколько платформ одновременно
- Запрос на адаптацию поста под конкретную площадку (TG, VK, IG, Pinterest, YouTube)
- Проактивный триггер: обнаружен перерыв в постинге > 3 дней
- Проактивный триггер: пост на одной площадке получил высокий охват — предложить кросс-пост
- Запрос на аналитику эффективности постов по площадкам
- Запрос на составление контент-календаря

## 3. Platform Support Matrix

### TIER 1: Full API — автоматическая публикация

| Платформа | Механизм | Статус |
|-----------|----------|--------|
| **Telegram** | CLI: `python3 scripts/tg-channel.py post @CHANNEL "текст" --photo /path` (Telethon userbot под капотом) | Работает |
| **VK** | CLI: `python3 scripts/vk-cli.py post --group GROUP_ID --text "текст" --photo /path` (VK API под капотом) | Работает |

Эти платформы поддерживают полный цикл: адаптация контента, публикация, получение ссылки на пост, сбор аналитики.

### TIER 2: Prepared Content — ручная публикация

| Платформа | Что готовит скилл | Что делает владелец |
|-----------|-------------------|---------------------|
| **Instagram** | Адаптированный текст + оптимизированное фото (1080x1080 или 1080x1350) + хэштеги (до 30) + подпись + CTA | Публикует вручную через приложение или через будущую Meta Graph API интеграцию |
| **Pinterest** | Описание пина (до 500 символов) + оптимизированное фото (1000x1500) + предложение доски + ключевые слова + alt-текст | Публикует вручную через Pinterest или через будущую API интеграцию |
| **YouTube** | Заголовок + описание (до 5000 символов) + теги (до 500 символов) + миниатюра + timestamps + конечные заставки (текст) | Загружает видео и вставляет метаданные вручную через YouTube Studio |

**ВАЖНО:** Скилл НЕ публикует контент на платформы Tier 2. Он готовит полностью оптимизированный пакет контента и сохраняет его в папку `{{OUTPUT_DIR}}/prepared/{{PLATFORM}}/{{DATE}}/`. Владелец забирает файлы и публикует самостоятельно.

## 4. Core Workflow

### Phase 1: Input & Validation

1. Принять исходный контент: текст + медиа (фото/видео) + список целевых площадок.
2. Если площадки не указаны — использовать все enabled-платформы из `platforms.json`.
3. Проверить наличие токенов/сессий для Tier 1 площадок. Если токен отсутствует — предупредить и пропустить площадку.
4. Проверить наличие медиа: если целевая площадка визуальная (IG, Pinterest) и фото отсутствует — остановить и запросить.

### Phase 2: Content Adaptation

5. Для каждой площадки вызвать скилл `social-content` для адаптации текста (см. раздел 5 — Platform-Specific Rules).
6. Вызвать `image-processing` для оптимизации медиа под размеры каждой площадки.
7. Сгенерировать площадко-специфичные метаданные: хэштеги, ключевые слова, alt-текст.

### Phase 3: Publication (Tier 1) / Preparation (Tier 2)

8. **Tier 1 — Telegram:** `python3 scripts/tg-channel.py post @{{CHANNEL_ID}} "текст" --photo /path`. Получить message_id и ссылку.
9. **Tier 1 — VK:** `python3 scripts/vk-cli.py post --group {{VK_GROUP}} --text "текст" --photo /path`. Получить post_id и ссылку.
10. **Tier 2 — Instagram/Pinterest/YouTube:** Сохранить подготовленный контент в `{{OUTPUT_DIR}}/prepared/{{PLATFORM}}/{{DATE}}/`:
    - `text.md` — адаптированный текст с хэштегами
    - `media/` — оптимизированные изображения
    - `metadata.json` — все метаданные (теги, описания, рекомендации)
    - `instructions.md` — пошаговая инструкция для ручной публикации
11. Уведомить владельца о готовности Tier 2 контента через Telegram DM.

### Phase 4: Report & Analytics

12. Сформировать отчёт:
```json
{
  "posts": [
    {"platform": "telegram", "tier": 1, "status": "published", "url": "https://t.me/...", "posted_at": "2026-04-03T14:30:00Z"},
    {"platform": "vk", "tier": 1, "status": "published", "url": "https://vk.com/wall...", "posted_at": "2026-04-03T14:30:05Z"},
    {"platform": "instagram", "tier": 2, "status": "prepared", "path": "/output/prepared/instagram/2026-04-03/", "posted_at": null},
    {"platform": "pinterest", "tier": 2, "status": "prepared", "path": "/output/prepared/pinterest/2026-04-03/", "posted_at": null}
  ]
}
```
13. Записать результаты в `{{ANALYTICS_DIR}}/posts_log.jsonl` для последующего анализа.

## 5. Platform-Specific Rules

### Telegram
- Длина: до 4096 символов (оптимально 500-1500)
- Форматирование: Markdown (жирный, курсив, моно, ссылки)
- Эмодзи: умеренно, 3-5 на пост, в начале абзацев
- CTA: в конце поста, одна чёткая ссылка или призыв
- Медиа: фото до 10 МБ, видео до 50 МБ, без автоматического сжатия
- Хэштеги: 2-5 в конце поста

### VK
- Длина: до 15895 символов (оптимально 1000-3000)
- Форматирование: wiki-разметка или plain text
- Стиль: более развёрнутый, повествовательный
- CTA: ссылки в тексте + кнопка «Подробнее»
- Опросы: использовать при вовлекающем контенте
- Хэштеги: 5-10, в том числе навигационные (#brand_рубрика)
- Медиа: прикреплять как вложения через API

### Instagram (Tier 2 — prepared only)
- Длина подписи: до 2200 символов (оптимально 150-300 для ленты)
- Визуал-first: пост БЕЗ фото не готовить
- Хэштеги: 20-30, смесь высокочастотных и нишевых, в первом комментарии
- Размеры фото: 1080x1080 (квадрат) или 1080x1350 (портрет, больше места в ленте)
- Alt-текст: всегда генерировать для каждого изображения
- CTA: вопрос в конце для комментариев, ссылка в шапку профиля

### Pinterest (Tier 2 — prepared only)
- Описание пина: до 500 символов, ключевые слова в первых 50
- Размер изображения: 1000x1500 (вертикальный 2:3)
- Текст на пине: крупный заголовок (если применимо)
- Ключевые слова: 5-10 релевантных для поисковой выдачи Pinterest
- Доска: предложить название доски на основе контента
- Alt-текст: обязателен, с ключевыми словами

### YouTube (Tier 2 — prepared only)
- Заголовок: до 100 символов, ключевое слово в первых 60
- Описание: до 5000 символов, структура: краткое содержание + timestamps + ссылки + хэштеги
- Теги: до 500 символов суммарно, от узких к широким
- Миниатюра: подготовить текст и рекомендации для дизайна (1280x720)
- Конечные заставки: текст для 2 слотов (подписка + рекомендованное видео)

## 6. Proactive Behaviors

Все proactive behaviors работают **при вызове скилла** (Claude inline). Автоматический мониторинг через cron — TODO (Phase 3).

### Posting Gap Detection (при вызове скилла)
- При запросе о постинге — проверить интервалы между публикациями для каждой площадки.
- Если перерыв > 3 дней: «Не было постов в {{PLATFORM}} уже {{N}} дней. Подготовить публикацию из последних товаров/новостей?»
- Предложить конкретный контент на основе доступных данных (каталог, новости, предыдущие популярные посты).

### Cross-Platform Amplification (при вызове скилла)
- Когда оператор спрашивает про аналитику — проверить метрики Tier 1 постов.
- Если пост получил 2x+ от среднего охвата: «Пост в {{PLATFORM}} получил {{VIEWS}} просмотров (в 2 раза больше среднего). Опубликовать адаптированную версию в {{OTHER_PLATFORMS}}?»

### Optimal Timing (при вызове скилла)
- При планировании поста — анализировать историю постов и их охват по времени суток (если данные есть в `posts_log.jsonl`).
- Предлагать оптимальное время: «По аналитике, лучшее время для VK: 18:00-20:00 (охват +35%). Запланировать публикацию на это время?»

### Content Calendar (по запросу оператора)
- По запросу предлагать расписание постов на неделю.
- Учитывать: баланс площадок, рекомендуемую частоту (TG: 3-5/нед, VK: 2-4/нед, IG: 3-7/нед), тематическое разнообразие.
- Формат: таблица с датами, площадками, темами и статусом подготовки.

### Content Repurposing (по запросу оператора)
- По запросу анализировать архив постов.
- Выявлять «вечнозелёный» контент с высокими метриками.
- Предлагать: «Пост от {{DATE}} набрал {{METRIC}} — обновить и переопубликовать? (прошло {{N}} месяцев)»

## 7. Self-Learning (Feedback Loops)

### Tracked Metrics
- **Reach:** просмотры (TG), охват (VK), показы (IG, Pinterest, YouTube)
- **Engagement:** лайки, комментарии, репосты/шеры, сохранения
- **Conversion:** клики по ссылкам, переходы на сайт
- **Timing:** время публикации → охват (корреляция)

### Learning Mechanisms

**Style & Format Optimization:**
- Сравнивать метрики постов с разными форматами (длинный vs короткий, с вопросом vs без, с эмодзи vs без).
- Записывать инсайты в `{{ANALYTICS_DIR}}/learnings.jsonl` (Claude создаёт файл при первом инсайте через Write tool):
```json
{"date": "2026-04-03", "insight": "Посты с вопросом в конце получают +40% комментариев в TG", "confidence": 0.75, "sample_size": 12}
```
- Применять накопленные инсайты при адаптации следующих постов.

**Timing Optimization:**
- Строить тепловую карту: день недели x час → средний охват.
- Обновлять рекомендации по оптимальному времени каждые 2 недели.
- Хранить в `{{ANALYTICS_DIR}}/timing_heatmap.json` (Claude создаёт/обновляет при запросе аналитики).

**Template Evolution:**
- Отслеживать, какие шаблоны описаний (для YouTube, Pinterest) приводят к большему вовлечению.
- Хранить топ-5 шаблонов по каждой платформе в `{{ANALYTICS_DIR}}/top_templates.json` (Claude создаёт при наличии данных, min 5 постов).
- Предлагать проверенные шаблоны при создании нового контента.

**A/B Insights:**
- Фиксировать естественные A/B тесты (похожий контент, разное оформление).
- Сообщать владельцу: «Посты с числами в заголовке получают +25% кликов (на основе 8 постов)».
- Минимальный sample_size для инсайта: 5 постов. До этого — пометка «предварительно».

## 9. Anti-Patterns

1. **НЕ публиковать одинаковый текст на все площадки.** ВСЕГДА адаптировать под формат, стиль и аудиторию каждой платформы. Копипаст = потеря охвата.

2. **НЕ постить без медиа на визуальных площадках.** Instagram и Pinterest без фото — не готовить. Запросить визуал у владельца.

3. **НЕ игнорировать лимиты API.** VK: 50 постов/день на группу. TG: без жёстких лимитов, но > 20 постов/день = мьют от подписчиков. Отслеживать и предупреждать.

4. **НЕ постить на площадки без настроенных токенов.** Если токен/сессия отсутствует или истёк — предупредить агента, пропустить площадку, предложить переключить в Tier 2 режим (подготовка).

5. **НЕ заявлять автоматическую публикацию для Tier 2 площадок.** Instagram, Pinterest, YouTube — только подготовка контента. В отчёте статус `"prepared"`, не `"published"`. В сообщении владельцу: «Контент для IG подготовлен в папке X. Опубликуйте вручную.»

6. **НЕ игнорировать оптимальное время.** Если есть данные аналитики — предлагать отложенную публикацию. Не публиковать в 3:00 ночи, если пик аудитории в 18:00.

7. **НЕ постить без проверки.** Перед публикацией в Tier 1 — показать превью агенту/владельцу. Ожидать подтверждение, если контент содержит ссылки, цены или промо-код.

8. **НЕ накапливать данные без анализа.** Аналитика без инсайтов бесполезна. Каждые 2 недели формировать краткий отчёт с выводами и рекомендациями.

## 10. Tools & Scripts

### Что делает Claude (inline, без скриптов)
- Адаптация текста под каждую площадку (вызывает `social-content` скилл)

- Tier 2 контент-подготовка (текст + инструкции → папка)
