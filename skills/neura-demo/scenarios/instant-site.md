# Сценарий 1: Сайт за 2 минуты

## Контекст
Самый сильный демо-сценарий. Собеседник называет свой бизнес — через 2-3 минуты открывает рабочий сайт на своём телефоне.

## Зависимые скиллы (прочитать перед выполнением)
1. `copywriting` — для текстов лендинга
2. `ui-ux-pro-max` — для дизайн-системы
3. `frontend-design` — для уникального визуала
4. `regru` — для деплоя (FTP на REG.ru)

## Алгоритм выполнения

### Шаг 1: Сбор контекста (10 сек)
Спросить кратко (или извлечь из голосового):
- Что за бизнес?
- Кто клиент?
- Что должен сделать посетитель сайта?

Если информации мало — додумать на основе ниши. В демо-режиме НЕ задавать много вопросов.

### Шаг 2: Генерация (60-90 сек)
Создать одностраничный лендинг:

**Структура:**
1. **Hero** — заголовок + подзаголовок + CTA
2. **Проблема** — 3 боли клиента
3. **Решение** — что предлагает бизнес
4. **Преимущества** — 3-4 карточки
5. **Социальное доказательство** — отзывы / цифры
6. **CTA** — финальный призыв

**Стек:** React + Vite + Tailwind (если есть время) или чистый HTML + Tailwind CDN (для скорости)

**Визуал:**
- Минималистичный neo-brutalist стиль
- Тёмная тема с акцентным цветом
- Крупная типографика
- Анимации при скролле (IntersectionObserver)
- Мобильная адаптация обязательна (демо на телефоне!)

### Шаг 3: Деплой (30-60 сек)
Использовать скилл `regru`:
- Деплой через FTP на поддомен demo.ceremoneymeister.ru или аналогичный
- Генерация .htaccess для SPA
- Проверка доступности

## Советы для максимального wow

- Использовать HTML + Tailwind CDN для максимальной скорости (не ждать npm install)
- Заготовить поддомен demo.ceremoneymeister.ru заранее
- Если деплой не получается — показать локальный скриншот + код
- Акцент на мобильную версию — собеседник смотрит на телефоне
- Добавить анимации появления секций — это впечатляет
